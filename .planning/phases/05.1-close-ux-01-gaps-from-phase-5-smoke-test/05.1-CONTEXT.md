**Gathered:** 2026-04-21

<domain>
## Phase Boundary

Close the four gaps surfaced by the Phase 5 Chrome smoke test (see `.planning/phases/05-student-experience-polish/05-VERIFICATION.md` "Gaps Found (2026-04-21) — route to Phase 05.1"):

- **Gap A** — NB adjective neuter-declension defects (surfaced by `blå_adj.declension.positiv.noytrum = "blåt"`; should be `"blått"`)

- **Gap C** — Gender-rule explain copy is generic; needs to name the target gender (e.g., "`by` er hankjønn")

The Phase 5 render layer is correct — this phase closes the gap between "correct render" and "end-to-end student-friendly." UX-01 stays `Pending` in REQUIREMENTS.md until this phase lands AND the smoke test passes AND the dyslexia-persona proxy re-reviews updated COPY-REVIEW.md.

New capabilities (English locale, dialect-mode-switch UI control, comprehensive ISO-639 language list, etc.) belong in future phases.

## Implementation Decisions

### Phase structure

- All four gaps (A, B, C, D) land in a single 05.1 phase — one release, one smoke test, one verification document revision.
- **Wave 1 parallel:** A, B, C, D execute concurrently. Cross-repo plans (A, B → `papertek-vocabulary`) and extension-side plans (C, D → `leksihjelp/extension/`) have zero file overlap.
- **Wave 2:** `sync-vocab` pulls A+B data into the extension, then the Chrome smoke test runs once against all four gaps.
- Ship as a GitHub Release after 05.1 — version bumps in `extension/manifest.json`, `package.json`, and `backend/public/index.html` per the CLAUDE.md release workflow.

### UX-01 close gate

- UX-01 marked complete in REQUIREMENTS.md only when **all three** hold:
  1. All four gap-closure plans merged.
  2. Chrome smoke test PASS across the five Phase-5 human-verification scenarios, re-run against the new copy and the new dialect-mix rule.
  3. Dyslexia-persona proxy re-reviews updated COPY-REVIEW.md (Gap C adds gender-naming copy, Gap D adds new dialect-mix copy — neither exists in the 2026-04-20 review).

### Gap A — `blå_adj` neuter fix + NB/NN adjective-declension audit

- **Fix site:** `papertek-vocabulary` — this is the source of truth. Leksihjelp consumes via `npm run sync-vocab`. Do not add a client-side workaround.
- Fix `vocabulary/lexicon/nb/adjectivebank.json:1624` — `"noytrum": "blåt"` → `"noytrum": "blått"`. Investigate the enrichment generator path (`_generatedFrom` metadata) to understand whether the defect originated upstream and fix there too if so.
- Write `scripts/validate-adj-declensions.js` in `papertek-vocabulary`: lint pass over every NB + NN adjectivebank entry. Assert each has a complete `declension.positiv` object. Flag entries where the `noytrum` form fails regular-adj morphology (base + `t` for consonant-final roots, base + `tt` for monosyllabic vowel-ending roots). Whitelist known irregulars explicitly. Report defects; fix each one found during 05.1.
- Apply the same audit to NN adjectivebank in parallel.
- One-time cleanup during 05.1. CI gate wiring in `papertek-vocabulary` is deferred (captured in Deferred Ideas).

### Gap B — Missing language/nationality nouns + new bank types

- **Schema change (cross-repo):** introduce two new bank types in `papertek-vocabulary` — `languagesbank` (language names: norsk, nynorsk, bokmål, engelsk, tysk, …) and `nationalitiesbank` (nationality adjectives/demonyms: nordmann, svenske, dansk, …). Additive change; existing bank consumers (webapps, nativeapps) are not impacted.
- Update `sync-vocab.js` to pull the new banks into `extension/data/*.json`.
- Update the `BANK_TO_POS` seam + `flattenBanks` code path so the new banks flow into `validWords` and into the dictionary-popup search index.
- **Targeted hand-authored list (~20 entries) for 05.1**, both NB and NN:
  - Languages: `norsk`, `nynorsk`, `bokmål`, `engelsk`, `tysk`, `spansk`, `fransk`, `italiensk`, `portugisisk`, `russisk`, `arabisk`, `kinesisk`, `japansk`, `polsk`, `ukrainsk`, `samisk`, `islandsk`, `færøysk`, `svensk`, `dansk`.
  - Nationalities: corresponding demonyms where grammatically distinct.
- Comprehensive ISO-639 + country-adjective list is deferred (see Deferred Ideas).

### Gap C — Gender explain copy names the target gender

- **Copy shape** (three beats, NB + NN symmetric):
  - NB: `<em>Et</em> kan være feil kjønn — <em>by</em> er hankjønn. Prøv <em>En</em>.`
  - NN: `<em>Eit</em> kan vere feil kjønn — <em>by</em> er hankjønn. Prøv <em>Ein</em>.`
- **Gender-label vocabulary:**
  - NB: `hankjønn` / `hunkjønn` / `intetkjønn`
  - NN (canonical): `hankjønn` / `hokjønn` / `inkjekjønn`
- **Label source:** `extension/i18n/strings.js`. Keys: `gender_label_m`, `gender_label_f`, `gender_label_n` in both `nb` and `nn` locale blocks. `nb-gender.js` resolves the label through `__lexiSpellCore.getString(key, lang)` (or equivalent helper) — the rule file does NOT embed the Norwegian strings directly. This keeps the i18n pattern consistent with Phase 5's `pred_vis_flere` / `settings_spellcheck_alternates_*` and makes future locale additions a strings-file edit rather than a rule-logic edit.
- **New finding field:** `finding.noun_display` carries the cased display form of the mismatched noun (e.g., `Byen`, not `byen`). `nb-gender.js:check` emits it on the result object alongside the existing `original` / `fix` fields.
- **Edge-case fallbacks:**
  - If `vocab.nounGenus.get(noun)` returns undefined, fall back to the current two-beat copy (no gender-class named).
  - Phase 4 feminine-as-common-gender acceptance (NB `en bok` stays non-flagged via the `actual === 'f' && articleTok.word === 'en'` guard at nb-gender.js:53-56) is preserved.
  - `escapeHtml` wraps all three interpolations: `original`, `noun_display`, `fix`.

### Gap D — Reverse `sisterValidWords` tolerance + new dialect-mix rule

**Seam disposition:**
- Keep `vocab.sisterValidWords` populated at the seam (vocab-seam-core.js:527+ and vocab-seam.js:272) — it's the data feed the new rule needs.
- **Remove** the cross-dialect early-exit in `nb-typo-curated.js:48` and `nb-typo-fuzzy.js:126`.
- **Keep** the sister-dialect lookup in `nb-codeswitch.js:55` — codeswitch's density window uses it as a "this token is the OTHER Norwegian" signal for scoring, which is orthogonal to the flag/no-flag decision this phase flips.

**New rule — `extension/content/spell-rules/nb-dialect-mix.js`:**
- Single file, `languages: ['nb', 'nn']`, lang-aware copy (matches the convention of other rules in `spell-rules/` that use `nb-` prefix while handling both dialects).
- **Priority: 30.** Slots between `nb-sarskriving.js` (25) and `nb-typo-curated.js` (40) so dialect-mix wins overlap against both typo rules (lower priority number = wins `dedupeOverlapping`).
- **Guard:** fires only when `token ∈ vocab.sisterValidWords` AND `token ∉ vocab.validWords`. Words valid in both dialects (hus, er, og, …) never fire.
- **Respects `ctx.suppressed`** — codeswitch and propernoun-guard can suppress dialect-mix the same way they suppress typo-curated / typo-fuzzy / sarskriving today.
- **Emits `finding.fix`** when a curated cross-dialect equivalent is known; flag-only otherwise.

**Cross-dialect mapping:**
- Curated JS constant `CROSS_DIALECT_MAP` inside the rule file for 05.1. ~30-80 high-confidence pairs (both directions):
  - NB → NN: `ikke/ikkje`, `jeg/eg`, `noen/nokon`, `bare/berre`, `meget/særs`, `hvem/kven`, `hva/kva`, `hvor/kvar`, `hvorfor/kvifor`, `hvordan/korleis`, `mye/mykje`, …
  - NN → NB: inverse of the above where bidirectional, plus NN-only forms.
- Cross-dialect shape: `Map<string, string>` keyed by the sister-dialect token, value is the current-dialect equivalent. Two maps (one per direction) or one bi-directional keyed by `{lang}:{word}` — planner picks the shape.
- Papertek-vocabulary migration of this map is deferred (see Deferred Ideas).

**Copy (explain callable returns `{nb, nn}`):**
- With fix (NB doc, student typed NN `ikkje`): `<em>ikkje</em> er nynorsk — prøv <em>ikke</em>.`
- With fix (NN doc, student typed NB `jeg`): `<em>jeg</em> er bokmål — prøv <em>eg</em>.`
- No-fix fallback (NB doc, sister-dialect word without a curated equivalent): `<em>ikkje</em> er nynorsk. Skriv du bokmål?`
- No-fix fallback (NN doc): `<em>jeg</em> er bokmål. Skriv du nynorsk?`
- Every interpolation escapeHtml-wrapped.

**Fixture handling:**
- Invert Phase 4 dialect-tolerance fixture cases in-place. Find every case in `fixtures/nb/typo.jsonl` / `fixtures/nn/typo.jsonl` / `fixtures/nb/clean.jsonl` / `fixtures/nn/clean.jsonl` that encodes "sister-dialect token → no finding" (e.g., `nb-typo-dialect-ikkje-001`) and flip to expected `{rule_id: 'dialect-mix', fix: 'ikke', …}`. Keep the fixture IDs; rename only if the original ID materially misrepresents the new expectation.
- Add new fixture cases covering: with-fix path, without-fix fallback path, `ctx.suppressed` interaction (dialect-mix inside a codeswitch-quoted block does not fire), NN→NB symmetry.

**SC-05 särskriving threshold gate:**
- Expected to be unaffected — `nb-sarskriving.js` doesn't consume `sisterValidWords`. Sanity-check `npm run check-fixtures` post-land confirms särskriving P/R/F1 stays above Phase 4's thresholds. No threshold edits planned.

<specifics>
## Specific Ideas

- User domain insight (2026-04-21, already captured in `memory/project_nb_nn_no_mixing.md`): **NB and NN are two distinct official written standards of Norwegian and may not be mixed in a single document.** Analogy user offered: "it is like you shouldn't accept German words when writing bokmål or nynorsk." This directly reverses Phase 4's dialect-tolerance policy. The new rule operationalizes the insight: cross-standard tokens are student errors, not dialect choice.
- User-proposed copy for Gap C: `"'Et' kan være feil kjønn — 'by' er hankjønn. Prøv 'En'."` — three beats, names the gender classification, teaches the student the rule (not just the fix).
- Tone consistency with Phase 5 COPY-REVIEW.md's dyslexia-persona proxy rules: one short sentence, no jargon, not accusatory, NN morphology correct (no "være/ikke/bare/hører" contamination in NN copy).
- All copy uses `<em>` spans for emphasis on user-facing tokens — matches the Phase 5 `renderExplain` template pattern and the already-verified XSS-safety contract.

</specifics>

<deferred>
## Deferred Ideas

- **English locale for explain copy.** Extend the `explain()` callable contract from `{nb, nn}` to include `en` (or a generalized lang-keyed object). Requires updating all five popover-surfacing rules' copy, extending `renderExplain`'s lang routing in `spell-check.js`, and translating COPY-REVIEW.md entries. Future phase — English is not the primary audience per PROJECT.md (Norwegian students learning foreign languages).
- **Migrate `CROSS_DIALECT_MAP` from curated JS to `papertek-vocabulary` data.** The map lives inside `nb-dialect-mix.js` for 05.1. A future phase can promote it to `papertek-vocabulary/vocabulary/cross-dialect/nb-nn.json`, exposed via the vocab seam, benefiting webapps + nativeapps too.
- **Comprehensive language/nationality list.** 05.1 ships a targeted ~20-entry hand-authored list. A follow-up phase can expand to a canonical ISO 639-derived languages set + country-adjective pairs using the new `languagesbank` / `nationalitiesbank` schema.
- **CI gate for adj-declension validator in `papertek-vocabulary`.** 05.1 runs the validator once as one-time cleanup. A future phase can wire it into `papertek-vocabulary`'s CI to catch regressions upstream before they reach consumers.
- **Fixture canaries in Leksihjelp for cross-repo data regressions.** A "`blått` must be in validWords" fixture would catch if sync-vocab ever re-introduces the defect. Belt-and-braces for Gap A; not needed if upstream validator is trustworthy.
- **Dialect-mode-switch control surface.** The no-fix copy "`jeg` er bokmål. Skriv du nynorsk?" currently only explains. A future phase could add a "switch to NN mode" quick-action in the popover so the student keeps the word and updates the document's dialect setting.
- **`finding.subject` / `finding.target` generic field.** Gap C uses `finding.noun_display` (specific to gender rule). If other rules later need to name a target token, a generic field could be introduced and existing rules migrated. Not needed now.

</deferred>
