<task type="auto" tdd="true">
  <name>Task 1: Implement vocab-updater.js with startup check + atomic refresh</name>
  <files>extension/background/vocab-updater.js, extension/background/service-worker.js, extension/background/vocab-updater.test.js</files>
  <behavior>
    - checkForUpdates() with all-fresh cache: returns {de:'fresh', es:'fresh',...}; emits no message.
    - checkForUpdates() with one stale: returns {de:'stale',...}; emits {type:'lexi:updates-available', langs:['de']}.
    - refreshLanguage('de'): fetches new bundle, atomically writes cache (no torn read possible), then calls swapIndexes; on schema-mismatch, leaves cache untouched and emits 'lexi:hydration' state='error'.
    - refreshAll runs sequentially; per-lang failure does not abort the rest.
  </behavior>
  <action>Create `extension/background/vocab-updater.js`:
- Import from `../content/vocab-store.js`: `API_BASE`, `getCachedRevisions`, `fetchBundle`, `putCachedBundle`.
- Implement `checkForUpdates()`:
  - Fetch `${API_BASE}/revisions`.
  - For each language present in cache (from getCachedRevisions), compare local vs server revision.
  - Status: 'fresh' if equal, 'stale' if differ. (Languages with no cache → 'missing'; let bootstrap handle them — do NOT emit update notice.)
  - If any 'stale', emit `chrome.runtime.sendMessage({type:'lexi:updates-available', langs:[...stale]})`.
  - Return the per-language status map.
- Implement `refreshLanguage(lang)`:
  - Call `fetchBundle(lang)` (no `If-None-Match` — we want the new bundle).
  - On `status: 200`: write the new payload via `putCachedBundle` (the IndexedDB put is atomic at the entry level — the existing entry is replaced in one transaction; readers either see old or new, never partial). Then dynamic-import the seam (or message a content-script bridge) to call `swapIndexes`. The seam's swap path also handles the next page-load case (next content-script load reads the updated cache fresh).
  - Emit `{type:'lexi:refresh-done', lang}`.
  - On `status: 'schema-mismatch'`: emit `{type:'lexi:hydration', lang, state:'error'}`. Do NOT touch the cache.
- Implement `refreshAll(langs)`: for-of loop calling refreshLanguage; catch per-lang errors so one failure doesn't abort.
- Wire into service-worker.js:
  - On `chrome.runtime.onStartup`, call `checkForUpdates()` (fire-and-forget; 30s timeout).
  - Add a message handler for `{type:'lexi:refresh-now', langs}` → `refreshAll(langs)`.
- Test file mocks fetch + vocab-store + chrome.runtime; asserts the four behaviors above.

CRITICAL: Atomic replacement is the IndexedDB transaction in `putCachedBundle`. Do not delete the old entry first. Active content scripts that already cached an indexes object in memory continue using it until `swapIndexes` runs (or until the next page navigation reloads the content script and re-reads the cache). That's the "activate on next page load" behavior the requirement specifies.</action>
