<objective>
Rewrite the vocab-store.js IndexedDB adapter and vocab-seam hydration so that:
- The cache hits the v1 endpoints (`/api/vocab/v1/bundle/{lang}`, `/api/vocab/v1/revisions`).
- Each cache entry stores `{schema_version, revision, fetched_at, payload}`.
- Vocab-seam reads from IndexedDB first; on miss, falls back to the bundled NB baseline (which today is `extension/data/nb.json` — plan 03 will shrink it).
- Hydration is async and non-blocking: spell-check + word-prediction render against baseline indexes immediately; when full data arrives, indexes rebuild in the background and swap atomically.
- A `schema_version` mismatch on a downloaded payload preserves the prior cache entry and emits a `lexi:schema-mismatch` diagnostic event the popup can show under "Developer view".

Purpose: Foundational data layer for plans 03–05. Without this, bootstrap, update detection, and migration have nothing to talk to.
Output: Updated vocab-store.js + vocab-seam.js with deterministic baseline-first behavior and schema gating.
</objective>
