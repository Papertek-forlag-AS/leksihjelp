<task type="auto">
  <name>Task 2: Write lockdown adapter contract documentation</name>
  <files>.planning/lockdown-adapter-contract.md</files>
  <action>Create `.planning/lockdown-adapter-contract.md`. Cover:
- **Why:** Lockdown imports leksihjelp via `scripts/sync-leksihjelp.js`. v3.0 removed bundled per-language vocab JSON; lockdown needs a story for how to feed `__lexiVocab` / `buildIndexes`.
- **Seam contract:** Document the public surface lockdown must satisfy:
  - `window.__lexiVocab` shape (the indexes object — point to vocab-seam-core.js as authoritative reference).
  - `vocab-store.js` exports lockdown can override or stub: `getCachedBundle(lang)`, `putCachedBundle`, `fetchBundle`, `getCachedRevisions`.
  - Hydration event surface: `chrome.runtime` messages `lexi:hydration`, `lexi:updates-available`, `lexi:refresh-done` (lockdown's chrome shim already handles `runtime.sendMessage`/`onMessage`).
- **Three implementation options for lockdown to pick from** (lockdown's own roadmap decides which):
  1. **Bundled vocab in lockdown's public dir** — simplest; lockdown ships its own `public/leksihjelp/data/{lang}.json` and overrides `getCachedBundle` to fetch via `chrome.runtime.getURL`-equivalent. Pros: zero new infra. Cons: lockdown ships full vocab on every release.
  2. **Lockdown-implemented IndexedDB bootstrap** — lockdown's loader script implements its own `vocab-store.js` shim that fetches from the same Papertek API at boot and caches in IndexedDB on the lockdown origin. Pros: parity with extension. Cons: CORS adjustments (lockdown's origin must be added to plan 23-01's CORS whitelist).
  3. **Hosted lockdown-side proxy** — lockdown deploys its own thin proxy that pre-fetches bundles and serves them with same-origin CORS; lockdown's vocab-store hits the proxy. Pros: no CORS friction; can layer auth. Cons: new infra to operate.
- **What this milestone delivers vs defers:**
  - Delivered: stable seam contract + documented options.
  - Deferred: actual lockdown-side implementation. Lockdown's own roadmap picks an option.
- **Coordination notes:** Per CLAUDE.md, lockdown ports leksihjelp fixes upstream before merging; lockdown's `npm install` re-syncs. So if lockdown picks option 2 and discovers a seam gap, the fix lands here, then lockdown re-syncs.

Keep it concise (~1-2 pages). This is a contract doc, not a tutorial.</action>
