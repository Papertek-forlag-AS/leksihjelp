  <action>
In nb-sarskriving.js check(), modify the compound detection logic to add a decomposition fallback when `compoundNouns.has(concat)` fails.

1. At the top of check(), extract decomposeCompound from the vocab bag:
   ```javascript
   const decompose = vocab.decomposeCompound; // Phase 17 COMP-07
   ```

2. Replace the simple `compoundNouns.has(prev.word + t.word)` check with:
   ```javascript
   const concat = prev.word + t.word;
   const isKnownCompound = compoundNouns.has(concat);
   const isDecomposable = !isKnownCompound && decompose &&
     (() => { const d = decompose(concat); return d && d.confidence === 'high'; })();
   
   if (isKnownCompound || isDecomposable) {
     out.push({ /* existing sarskriving finding */ });
   }
   ```

   Key points:
   - Only try decomposition when compoundNouns.has() fails (short-circuit)
   - Only flag when confidence === 'high' (both components must be known nouns)
   - SARSKRIVING_BLOCKLIST is checked BEFORE this code runs (existing flow unchanged)
   - The existing `prev.word.length >= 2 && t.word.length >= 2` guard still applies
   - Do NOT call decompose on every token pair — only when both pass blocklist and length guards

3. Do NOT change the rule's id, priority, severity, or explain function.
4. Do NOT add decomposed compounds to compoundNouns (STATE.md Pitfall 3/6).

Pitfall 3 from research: Some adjectives also appear in nounGenus (e.g., "god" as a noun). The SARSKRIVING_BLOCKLIST is the first gate and already includes common adjectives. Decomposition adds a second gate: BOTH components must be known nouns in nounGenus. If "stor" is not in nounGenus as a noun, "stor dag" won't decompose. If "stor" IS in nounGenus, the blocklist catches it. Belt and suspenders.
