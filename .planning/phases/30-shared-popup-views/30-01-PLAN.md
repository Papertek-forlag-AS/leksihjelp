<objective>
Refactor `extension/popup/popup.js` (3148 lines, monolithic) into four mountable view modules — dictionary, settings, pause, report — using explicit dependency injection. The extension popup must continue to behave **identically** for paying users; this is non-negotiable since popup.js is in production. Account/auth, payments, exam-toggle, vocab-refresh, "Skriv", "Pin", and "Simuler lærer-lås" stay in popup.js (extension-only) and are NOT mounted in lockdown later.

Purpose: Phase 30 establishes a single source of truth for the popup's user-facing views so the lockdown sidepanel (Plan 30-02) can mount the same modules with limited deps and stop drifting from the extension. This plan creates the contract; 30-02 consumes it.

Output: Four view modules with `mount{Name}View(container, deps)` signatures, view-relevant CSS extracted, popup.js wired to the new modules, view-module unit tests, and a `check-popup-deps` release gate that hard-fails if a view module reaches for implicit globals.
</objective>
