<objective>
Close Phase 2 with a release-gate script that enforces the 10 MiB extension zip ceiling (ROADMAP success criterion #4). The existing packaged zip was 10.26 MiB BEFORE Phase 2 started — already over the publicly-stated 10 MB cap — and research Pitfall 3 flagged this as a blocking concern. After Plan 02-01 adds freq-{nb,nn}.json (~100-200 KB gzipped combined), Plan 02-02 grows bigrams-{nb,nn}.json (~100 KB gzipped combined), and Plan 02-03 expands the typo bank inside data/*.json (pulling in ~30-50% more typo strings via sync-vocab), the zip will be heavier still. This plan adds the measurement tool AND the size-reducing action needed to bring the zip back under 10 MiB, then wires the script into CLAUDE.md's Release Workflow so future regressions are caught pre-release.

The size-reducing action is JSON minification of `extension/data/*.json` at package time. Research Open Question 1 concludes: start with JSON minification (the data/*.json files are currently pretty-printed with 2-space indent; stripping that saves 15-25% per file — at 35 MB uncompressed across the data directory, that's ~5-8 MB saved uncompressed or roughly 1-2 MB inside the zip). This is invisible to consumers because the extension reads the files as parsed JSON either way. If minification alone is insufficient, the plan escalates to a decision checkpoint surfaced to the user (e.g. strip bundled audio/de/ and fetch on first use — that's a product decision, not a research call).

Purpose:
- Make success criterion #4 measurable and automated: "Total packaged extension zip size from `npm run package` stays within the 10 MB ceiling, verified in the release checklist."
- Prevent silent bundle-size creep from future vocab/data growth.
- Close the gap created by Plan 02-01 + 02-02 + 02-03 so Phase 2 doesn't ship a zip larger than the publicly-stated 10 MB promise.

This plan MUST run after the three Wave-1 plans are complete because it needs the combined output of all three data additions to measure the realistic packaged size. Waves 1 and 2 cannot parallelise on this plan — the measurement needs their output to be the final word.

Output:
- `scripts/check-bundle-size.js` — new Node CommonJS script that runs `npm run package`, measures the zip, prints per-directory byte breakdown, exits 0 if under 10 MiB and 1 if over.
- `package.json` gains a `"check-bundle-size": "node scripts/check-bundle-size.js"` script.
- `extension/data/*.json` files are minified at package time (either by a small modification to the `package` npm script, or by a pre-package step invoked from check-bundle-size.js) so zip size comes down.
- `CLAUDE.md` Release Workflow updated to include `check-bundle-size` as a mandatory step between `check-fixtures` and the version bump.
- The zip, after all Phase 2 data additions and minification, measures under 10,485,760 bytes.
</objective>

PART B — Create `scripts/check-bundle-size.js`:

Node CommonJS, zero deps. Uses the same style as `scripts/check-fixtures.js`:
```javascript
'use strict';
const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');
```

Constants:
- `CEILING_BYTES = 10 * 1024 * 1024; // 10 MiB = 10,485,760`
- `ZIP_PATH = path.join(__dirname, '..', 'backend', 'public', 'lexi-extension.zip')`

Flow:
1. Run `npm run package` via `execSync('npm run package', { cwd: path.join(__dirname, '..'), stdio: 'inherit' })`. If it throws, print a clear error and exit 1.
2. Stat the zip: `const size = fs.statSync(ZIP_PATH).size;`
3. Run `unzip -l` on the zip to get a listing with per-file compressed sizes. Parse the output — for each entry, assign it to its top-level directory (the first path segment). Accumulate sums per directory. `execSync('unzip -l ' + ZIP_PATH, { encoding: 'utf8' })` then regex-split lines.
4. Print the breakdown sorted by size descending, with human-readable MiB alongside raw bytes, for every directory ≥ 1 MiB. Lines like: `  data/     8,234,123 bytes  (7.85 MiB)`.
5. Print the total: either `PASS: 9.87 MiB (9,876,543 bytes) — under cap 10,485,760` or `FAIL: 10.51 MiB (11,012,344 bytes) — OVER cap 10,485,760 by 526,584 bytes`.
6. `process.exit(size <= CEILING_BYTES ? 0 : 1);`

Register both new npm scripts in `package.json`:
- Update `"package": "node scripts/package-extension.js"` (replaces the existing shell-only script).
- Add `"check-bundle-size": "node scripts/check-bundle-size.js"`.

After implementing both scripts:
