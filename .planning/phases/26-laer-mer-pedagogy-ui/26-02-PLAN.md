<objective>
Add a new release-workflow gate `check-pedagogy-shape` that enforces the pedagogy block contract anywhere a rule emits one. This is the structural counterpart to `check-explain-contract`: explain returns text, pedagogy carries structured teaching data, and both must stay valid as the popover (plan 26-03) reads them.

Purpose: Without this gate, a future rule could emit a `pedagogy` block missing `summary.nn` or with `case: 'wechselpräposition'` (typo) and the popover would render visibly broken text — but no other gate would catch it. This is exactly the "Phase 05.1 dialect-mix" class of regression that motivated the existing wiring gates.

Output:
- `scripts/check-pedagogy-shape.js` — loads all spell rules, finds the ones whose runtime `check()` produces findings with `.pedagogy`, validates each pedagogy block.
- `scripts/check-pedagogy-shape.test.js` — paired self-test using the same plant/restore pattern as check-explain-contract.test.js.
- Two new entries in package.json `scripts`.

This plan can land in parallel with 26-01 (no shared files). The gate stays informational (exit 0) until 26-01 wires de-prep-case to attach pedagogy.
</objective>

```js
{
  case: 'akkusativ' | 'dativ' | 'wechsel' | 'genitiv',  // REQUIRED, enum
  summary:     { nb: string, nn: string, en: string },   // REQUIRED, all three non-empty
  explanation: { nb: string, nn: string, en: string },   // REQUIRED, all three non-empty
  examples?: [                                           // OPTIONAL, but if present each entry needs:
    { correct: string, incorrect: string,
      translation: { nb, nn, en },
      note?: { nb, nn, en } }
  ],
  wechsel_pair?: {                                       // OPTIONAL, but required when case === 'wechsel'
    motion:   { sentence, translation, note },
    location: { sentence, translation, note }
  },
  colloquial_note?: { nb, nn, en },                      // OPTIONAL
  contraction?: { from: string, article: string }        // OPTIONAL
}
```

The gate validates EVERY pedagogy block emitted by EVERY rule, then exits 0/1 accordingly.

         * `vocab.nounGenus`: Map containing `Schule → 'f'`, plus a few others to make rules with broader expectations happy
         * `vocab.prepPedagogy`: Map containing `durch → { case: 'akkusativ', summary: {nb:'X',nn:'X',en:'X'}, explanation: {nb:'Y',nn:'Y',en:'Y'} }` (a known-valid block, so the rule's hookup gets exercised)
         * Other fields the existing check() functions touch — copy whatever check-explain-contract's fakeFinding-equivalent uses; if some rules need more, fall through gracefully (catch errors, treat as "no findings emitted" — the gate is interested in pedagogy SHAPE, not whether every rule fires under synthetic input).

    3. For every finding returned that has a `pedagogy` field, validate the shape:
       - `case` is one of ['akkusativ', 'dativ', 'wechsel', 'genitiv']
       - `summary` exists and `summary.nb`, `summary.nn`, `summary.en` are all non-empty strings
       - `explanation` exists with the same {nb,nn,en} non-empty triple
       - If `case === 'wechsel'`, `wechsel_pair.motion` and `wechsel_pair.location` must exist
       - If `examples` is present, each entry must have `correct: string`, `incorrect: string`, `translation: {nb,nn,en}` non-empty
       - If `colloquial_note` is present, must be `{nb, nn, en}` non-empty triple
       - If `contraction` is present, must have `from: string` and `article: string`
    4. On any violation: stderr.write a structured FAIL line with the rule id, file path, missing/wrong field, and exit 1. Mirror check-explain-contract's `fail()` helper exactly.
    5. If no rule emits any pedagogy finding (the pre-26-01 state), print `[check-pedagogy-shape] PASS: no rules emit pedagogy yet — informational` and exit 0. This lets the gate land before the rule hookup ships.
    6. Otherwise print `[check-pedagogy-shape] PASS: validated N pedagogy blocks across M rules` and exit 0.
