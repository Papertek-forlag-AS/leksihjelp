<objective>
Land the DE preposition pedagogy data inside the bundled extension and route it through the rule pipeline so a finding produced by `de-prep-case` carries the matching `pedagogy` block when one exists. This is the data-side and rule-side of the phase. The popover UI that consumes `finding.pedagogy` is built in plan 26-03.

Purpose: Without this plan, the popover-side work has nothing to render. The pedagogy data exists in papertek-vocabulary commits 664f2970 / 937ef4a2 / 7bdf6775 but is invisible to Leksihjelp until sync runs and the rule reaches for it.

Output:
- Verified or fixed `scripts/sync-vocab.js` so `pedagogy` survives the round trip (the current spread-based clean step looks like it should, but a regression hidden in the destructure-and-spread block must be ruled out).
- Re-synced `extension/data/de.json` containing the 34 enriched DE prep entries with pedagogy.
- A new `prepPedagogy` Map in `vocab-seam-core.buildIndexes` keyed by the surface forms students actually type (handles ä/ö/ü normalization and `_prep` suffix stripping).
- `de-prep-case.js` populates `finding.pedagogy` from that Map when fire-time lookup hits.
</objective>
