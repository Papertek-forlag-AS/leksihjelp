<feature>
  <name>predictCompound — compound word prediction from partial input</name>
  <files>extension/content/vocab-seam-core.js, test/phase-24-unit.test.js</files>
  <behavior>
    predictCompound(partial, nounGenus, lang) returns an array of compound suggestions.

    Algorithm:
    1. If partial.length < 4, return [].
    2. For splitPos from 3 to partial.length - 1:
       a. Extract left = partial.slice(0, splitPos)
       b. If nounGenus.has(left):
          - remainder = partial.slice(splitPos)
          - Try zero-fuge: scan nounGenus keys starting with remainder, for each match build full compound = left + key, verify via decomposeCompound, include if valid
          - Try each linker in LINKERS_BY_LANG[lang]: if remainder starts with linker, stripped = remainder.slice(linker.length), scan nounGenus keys starting with stripped, build compound = left + linker + key, verify via decomposeCompound
    3. Deduplicate by compound word. Cap at 10 results.
    4. Return array of { word: fullCompound, decomposition: decomposeResult }.

    Test cases:
    - 'chefsstu' with DE nounGenus containing 'chef','stuhl' -> suggests 'chefsstuhl' (fuge-s)
    - 'skoleg' with NB nounGenus containing 'skole','gutt' -> suggests 'skolegutt' (zero-fuge)
    - 'hverdagsm' with NB nounGenus containing 'hverdag','mas' -> suggests 'hverdagsmas' (fuge-s)
    - 'abc' -> [] (too short)
    - 'xyzabc' -> [] (unknown first component)
    - 'skole' alone (no partial second) -> [] (need at least 1 char of second component)
    - Results include decomposition object with parts, gender, confidence
  </behavior>
  <implementation>
    1. Add `predictCompound(partial, nounGenus, lang, decompFn)` inside the IIFE, after `decomposeCompound`.
    2. The function scans nounGenus keys for prefix matches on the remainder after splitting off a known left component.
    3. For efficiency, build a sorted array of nounGenus keys once (lazy, cached) so prefix scanning uses binary search or simple filter. Given typical nounbank sizes (~5K-15K entries), a simple `.filter(k => k.startsWith(prefix))` is fast enough for interactive use.
    4. Add bound closure to buildIndexes return: `predictCompound: (partial) => predictCompound(partial, nounGenus, lang, (w) => decomposeCompound(w, nounGenus, lang))`.
    5. Export `predictCompound` in the dual-export api object.
    6. Follow test pattern from test/phase-16-unit.test.js (plain Node.js + assert.strict).
  </implementation>
</feature>
