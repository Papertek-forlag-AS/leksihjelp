FR clitic rank table (inline in rule file per data-logic-separation philosophy):
```javascript
const FR_CLITIC_RANKS = {
  me: 1, te: 1, se: 1, nous: 1, vous: 1,
  le: 2, la: 2, les: 2,
  lui: 3, leur: 3,
  y: 4,
  en: 5,
};
```

Key guards from research:
- le/la/les are also articles — only treat as clitic when in pre-verbal window (not followed by noun)
- nous/vous can be subjects — stop collecting clitics when hitting a subject pronoun
- Skip imperative-affirmative (verb-hyphen-clitic constructions)

Create extension/content/spell-rules/fr-clitic-order.js following the self-registering IIFE pattern:

1. Register rule with id 'fr-clitic-order', languages ['fr'], priority 60.
2. Define FR_CLITIC_RANKS inline (see interfaces above). This is stable grammar-level data — inline per data-logic-separation philosophy (closed class, stable for centuries).
3. Define FR_SUBJECT_PRONOUNS = SUBJECT_PRONOUNS.fr reference (or inline Set if simpler: new Set(['je', 'tu', 'il', 'elle', 'on', 'nous', 'vous', 'ils', 'elles'])).
4. In check(ctx): iterate through sentences. For each sentence:
   a. Scan for finite verb tokens (ctx.getTagged(i).isFinite).
   b. For each finite verb at index i, walk backwards collecting clitic tokens:
      - Start at i-1, walk back toward sentence start.
      - For each token: get lowercase word, check if in FR_CLITIC_RANKS.
      - GUARD: if token is 'le'/'la'/'les', check if the NEXT token (toward verb) is NOT a clitic and NOT the verb itself — if so, it's likely an article, stop collecting.
        Simpler: only treat le/la/les as clitic if token at (k+1) is either another clitic or the verb itself.
      - GUARD: if token is in FR_SUBJECT_PRONOUNS and is at sentence start or tagged isSubject, stop collecting (this is the subject, not a clitic).
      - GUARD: if token is a hyphen or follows a hyphen (imperative: "donne-le-moi"), skip this verb entirely.
      - If token is not a clitic and not a subject pronoun, stop collecting.
      - Collect { idx: k, rank, word } into array (unshift to maintain order).
   c. If 2+ clitics collected, check if ranks are in ascending order.
   d. If any rank[i] < rank[i-1] (descending = wrong order): flag the span from first clitic to last clitic. Suggestion: reorder clitics by ascending rank and join.
5. explain() returns {nb, nn, severity: 'warning'} explaining FR clitic order rule (me/te/se < le/la/les < lui/leur < y < en).
