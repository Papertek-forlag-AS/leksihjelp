## Confirmed Root Cause

The Papertek API at `https://papertek-vocabulary.vercel.app/api/vocab/v1/bundle/fr` serves FR `generalbank` from `vocabulary/lexicon/fr/generalbank.json` (865 keys, MISSING the 3 aspect meta entries) instead of `vocabulary/core/fr/generalbank.json` (866 keys, has them). The bundled `extension/data/fr.json` has all entries (sourced from `core/`), but `vocab-seam.js`'s `BUNDLED_LANGS = ['nb']` forces FR through the API path, skipping the bundled file.

Without the 3 aspect entries in `raw.generalbank`, `buildIndexes` produces empty `frAspectAdverbs.imparfait.phrases` and `null frAspectPedagogy`, so the `fr-aspect-hint` rule silently returns no findings — including the canonical UAT-EXT-01 sentence.

The Plan 38-01.1 Branch C fix (`nb-typo-fuzzy` elision strip) was a real-but-symptomatic improvement (correctly stopped NB-typo from false-firing on FR elided auxiliaries like `j'ai`), but it did NOT address why fr-aspect-hint stayed silent. That required this data-load fix.

Why upstream papertek-vocabulary fix was NOT chosen: cross-app blast radius (lockdown webapp + skriveokt-zero are also consumers); user explicitly constrained Plan 38-01.2 to leksihjelp-only — live staging testing on stb-lockdown.app for Saturday must not be invalidated. Defensive seam-side fix is the right tradeoff.
