# Phase 38 Plan 01.1: F38-1 fr-aspect-hint blocker fix Summary

Strip FR elision in nb-typo-fuzzy's cross-language verb-form guard so `j'ai`, `n'a`, `qu'as` and similar elided auxiliaries no longer leak past the suppression check. Closes F38-1 walker symptom (NB-typo firing on French tokens, mis-suggesting `j'ai → j'aime`) and lands the regression fixture `fr-aspect-pos-f38-1-canonical` to lock the bug path. Companion popup-plumbing hardening included; sidecar gap deferred. Release v2.9.19 created as draft pending re-walk.

## What was done

1. **RED fixture (Task 1, commit 744aec8):** Added `fr-aspect-pos-f38-1-canonical` row to `fixtures/fr/aspect-hint.jsonl` pinning the canonical UAT-EXT-01 Step 1 sentence `Pendant que je marchais, j'ai vu un chien.` Initial fixture (negative — `expected: []`) failed because `nb-typo-fuzzy` (multi-language) flagged `j'ai` and suggested `j'aime` — exactly the F38-1 walker symptom, reproduced deterministically in the Node runner. This proved the proximate cause is rule-side, not browser-only popup plumbing as initially hypothesised.

2. **GREEN fix (Task 2, commit 8214f4d):** Two surgical patches:
   - `extension/content/spell-rules/nb-typo-fuzzy.js:162-185` — added `ELISION_RE = /^(?:j|n|s|c|d|l|m|t|qu)'(.+)$/i` and `stripElision()` helper. `tokenIsForeignVerbForm()` now checks BOTH the literal token AND the elision-stripped form against `frAuxPresensForms` (which contains unelided bases `ai`, `a`, `as`, `est`, …). Mirrors the existing elision-detection pattern in `fr-aspect-hint.js:248`.
   - `extension/popup/popup.js:1289-1306` — moved the `chrome.runtime.sendMessage({type:'LANGUAGE_CHANGED'})` broadcast OUT of the try/catch wrapping `loadDictionary` + `loadGrammarFeatures`. Previously, a throw from those silently swallowed the broadcast, leaving content scripts on stale `currentLang`. Companion preventative fix.
   - Refined the fixture row to `expected: [{rule_id: 'fr-aspect-hint', start: 25, end: 32, suggestion: "j'ai vu"}]` matching the rule's actual output (rule fires correctly on the passé-composé compound; `Pendant que` is in the imparfait phrase set).
   - F38-1 frontmatter: `regression_fixture_id: fixtures/fr/aspect-hint.jsonl`, `status: closed`. Resolution section appended.

3. **Release plumbing (Task 3, commit c54caed):** All 14 release gates passed. `npm version patch` → 2.9.19; manually bumped `extension/manifest.json` and `backend/public/index.html` to match. `npm run package` produced `backend/public/lexi-extension.zip` (12.68 MiB). Tagged `v2.9.19`, pushed to origin, created GitHub Release **as DRAFT** with zip asset attached.

4. **Task 4 (re-walk):** Hard-paused — `verification_kind: human-browser-walk`. Awaits walker re-walk of UAT-EXT-01 Steps 1-4 against the v2.9.19 zip.

## Branch identification

Per plan's three candidate root causes:

- **Branch A (popup→seam plumbing):** Latent bug confirmed (LANGUAGE_CHANGED broadcast gated behind try/catch). Fixed preventatively but NOT the F38-1 critical path.
- **Branch B (silent hydration failure):** Not exercised — sidecar 404s are graceful, FR vocab loads via vocab-store.
- **Branch C (stale snapshot / rule-side leakage):** **The actual F38-1 root cause.** Multi-language `nb-typo-fuzzy` rule's F36-1 cross-language guard checked `vocab.frAuxPresensForms.has(t.word)` against the literal token. FR tokenizer emits `j'ai` (apostrophe included); `frAuxPresensForms` contains only unelided bases. Guard slipped on every elided auxiliary.

## Why all 12 release gates stayed green pre-fix

No fixture exercised an elided-auxiliary FR token in a context where `nb-typo-fuzzy`'s F36-1 guard was relied on. The Phase 36-02 INFRA-10 gate (`check-vocab-seam-coverage`) defends seam-shape, not token-shape. The new fixture row plugs this gap — future regressions of this class will now be caught by `check-fixtures` before release.

## Out of scope (queued)

- **Sidecar-pipeline gap (universal):** `freq-{lang}.json`, `pitfalls-{lang}.json`, `bigrams-{lang}.json` for FR/DE/ES, plus `pitfalls-nb.json`. All 404 in browser console. Per plan must_haves and data-logic separation philosophy: sidecar generation is upstream Papertek-API/sync-vocab territory. Queued as **38-01.2 candidate** follow-up.
- **fr-aspect-hint canonical-narrative false-positive:** `Pendant que je marchais, j'ai vu un chien` is grammatically correct French — subordinate imparfait + main passé composé is the canonical narrative pair. Current rule flags `j'ai vu` because `Pendant que` is in the imparfait phrase set. Refining the rule to skip subordinator-introduced clauses is a separate concern; future plan.
- **Centralising elision-strip:** Same regex pattern now lives in two places (`fr-aspect-hint.js:248` and `nb-typo-fuzzy.js:162`). Refactor into a shared util when a third call-site lands.
