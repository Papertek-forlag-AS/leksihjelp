  <files>extension/content/spell-rules/nb-v2.js</files>
  <action>
Add four guards to the nb-v2 check() function to eliminate false positives on legitimate SVO constructions:

**Guard 1 — Coordinate conjunctions:** Before flagging subject+verb at position i, scan tokens between range.start and i for coordinate conjunctions ("og", "men", "eller", "for" in NB; "og", "men", "eller", "for" in NN). If a coordinate conjunction is found and the subject pronoun immediately follows it (or follows with only 1 token gap), skip — the second clause of a coordination uses SVO order correctly.

Define COORD_CONJUNCTIONS:
```javascript
const COORD_CONJUNCTIONS = new Set(['og', 'men', 'eller', 'for']);
```

In the main loop, after `if (hasSubBefore) continue;`, add:
```javascript
// Guard: coordinate conjunction before subject → second clause SVO is correct
let hasCoordBefore = false;
for (let j = range.start; j < i; j++) {
  const jTagged = ctx.getTagged(j);
  if (jTagged && COORD_CONJUNCTIONS.has(jTagged.word)) {
    hasCoordBefore = true;
    break;
  }
}
if (hasCoordBefore) continue;
```

**Guard 2 — Question-initial (finite verb at position 0):** If the token at range.start is itself a finite verb (checked via knownPresens/knownPreteritum), the sentence is a yes/no question. Subject follows verb naturally in questions. Add this check right after the subordinate-clause skip block (around line 84):
```javascript
// Skip question-initial: if first word is a finite verb, subject+verb order is normal
const firstIsFinite = (ctx.vocab.knownPresens && ctx.vocab.knownPresens.has(firstWord)) ||
                       (ctx.vocab.knownPreteritum && ctx.vocab.knownPreteritum.has(firstWord));
if (firstIsFinite) continue;
```

**Guard 3 — Complement clause "det er":** The token pair "det er" after verbs like "sier/sier/vet/tror" etc. is a complement clause (implicit "at"). This is already partially handled by the subordinator check, but when "at" is omitted, "det er" appears in what looks like a main clause. Add: if the subject is "det" and the verb is "er" (or "var"), and the preceding token is a known finite verb, skip. Insert after the coordinate conjunction guard:
```javascript
// Guard: complement clause — "[finite verb] det er/var" pattern
if (tagged.word === 'det' && (nw === 'er' || nw === 'var')) {
  const prevTagged = i > range.start ? ctx.getTagged(i - 1) : null;
  if (prevTagged) {
    const pw = prevTagged.word;
    const prevIsFinite = (ctx.vocab.knownPresens && ctx.vocab.knownPresens.has(pw)) ||
                          (ctx.vocab.knownPreteritum && ctx.vocab.knownPreteritum.has(pw));
    if (prevIsFinite) continue;
  }
}
```

**Guard 4 — NN article "ein" disambiguation:** In NN, "ein" is both a subject pronoun and an indefinite article. When followed by a noun (not a verb that's also in the noun bank), it's an article. Check: if subject candidate is "ein" and the next token (the supposed verb) is also a known noun, skip. Add before the isDirectFinite check:
```javascript
// Guard: NN "ein" as article, not pronoun — if next word is a known noun, skip
if (lang === 'nn' && tagged.word === 'ein') {
  if (ctx.vocab.nounGenus && ctx.vocab.nounGenus.has(nw.toLowerCase())) continue;
}
```

After all four guards are added, run `npm run check-fixtures 2>&1 | grep "FAIL"` to verify the nb-v2 false positives on non-Phase-7 fixtures are eliminated. The remaining failures should only be co-fire expected-array mismatches on Phase 7 fixture files.
