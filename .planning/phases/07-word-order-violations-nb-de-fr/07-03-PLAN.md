**Create extension/content/spell-rules/de-verb-final.js:**

IIFE rule following the standard pattern:
- `id: 'de-verb-final'`, `languages: ['de']`, `priority: 67`, `severity: 'warning'`
- Uses `findFiniteVerb`, `findSubordinator`, `tokensInSentence` from `__lexiSpellCore`
- Define DE_SUBORDINATORS locally (same set as in core, but needed for clause-boundary scanning): `dass/weil/wenn/ob/obwohl/als/bevor/nachdem/damit/sodass/solange/sobald/seit/seitdem/während/indem/falls`
- Honor `ctx.suppressedFor.structural`

Detection algorithm per Research:
1. For each sentence, get token range via `tokensInSentence`
2. Find subordinator(s) in the sentence
3. For each subordinator found, define the subordinate clause:
   - Start: subordinator position + 1
   - End: next comma token, next period, or sentence end (Pitfall 4: if no comma, extend to sentence end)
4. Find finite verb in the subordinate clause
5. If finite verb is NOT the last content token before clause boundary → flag
   - "Last content token" = last token that isn't punctuation (period, comma, exclamation, question mark)
6. Suggestion: show the corrected word order with verb moved to end

`explain`:
- NB: `"I tyske bisetninger skal det bøyde verbet stå til slutt — flytt <em>${escapeHtml(finding.display)}</em> til slutten av bisetningen."`
- NN: `"I tyske bisetningar skal det bøygde verbet stå til slutt — flytt <em>${escapeHtml(finding.display)}</em> til slutten av bisetningen."`

**Create extension/content/spell-rules/fr-bags.js:**

IIFE rule:
- `id: 'fr-bags'`, `languages: ['fr']`, `priority: 68`, `severity: 'hint'` (P3 — meaning-shift ambiguity)
- Define BAGS_SET constant inside the IIFE (the ~40 forms from Research)
- Define FR_ARTICLES: `new Set(['le','la','les','un','une','des','l','du','au','aux'])`
- Honor `ctx.suppressedFor.structural`

Detection algorithm per Research:
1. For each token (index i), check if `token.word.toLowerCase()` is in BAGS_SET
2. If yes, check if preceding token (i-1) is a noun:
   - Check `ctx.vocab.nounGenus.has(prevToken.word.toLowerCase())`
   - Also check: if token at i-2 is an article (FR_ARTICLES), then i-1 is likely a noun even if not in nounGenus
3. If BAGS adjective follows a noun → flag the adjective token
4. Do NOT flag if BAGS adjective precedes a noun (correct placement)
5. Suggestion: "Flytt adjektivet foran substantivet" (Move adjective before noun)

`explain`:
- NB: `"BAGS-adjektiv (Beauty, Age, Goodness, Size) står vanligvis foran substantivet på fransk — flytt <em>${escapeHtml(finding.display)}</em> foran substantivet."`
- NN: `"BAGS-adjektiv (Beauty, Age, Goodness, Size) står vanlegvis framfor substantivet på fransk — flytt <em>${escapeHtml(finding.display)}</em> framfor substantivet."`
