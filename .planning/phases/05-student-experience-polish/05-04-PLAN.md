<objective>
Add the user-togglable "Vis alternative skriveforslag" setting to the extension popup's Settings tab — the single control mechanism for turning on UX-02's spell-check popover multi-suggest layout (which Plan 05 implements in spell-check.js). Follows the existing toggle precedent set by `setting-prediction` + `setting-darkmode` — same CSS classes, same `chromeStorageGet`/`chromeStorageSet` pattern, same place in the Settings tab flow. Default OFF to preserve today's low-noise popover behavior for first-time users.

This plan is Wave 2, parallel with Plans 02 + 03. It depends only on Plan 01 (for the three i18n keys: `settings_spellcheck_alternates_title`, `_toggle`, `_note`). It modifies only popup-surface files — zero content-script impact, so it has no file-ownership overlap with Plan 02 (rule files) or Plan 03 (word-prediction.js / content.css).

Purpose: ship the control surface for UX-02's popover multi-suggest so Plan 05's consumer (`spell-check.js`) has a real `chrome.storage.local.spellCheckAlternatesVisible` key to read + react to. Even before Plan 05 lands, the toggle is harmless — it writes a key no content script reads yet.

Output: new settings-group block in `extension/popup/popup.html` (after the existing `setting-prediction` group), new `initSettings` wiring in `extension/popup/popup.js` (reads storage on init, writes on change).
