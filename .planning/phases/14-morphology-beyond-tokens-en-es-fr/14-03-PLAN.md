<objective>
Implement MORPH-02 (FR/ES opaque-noun gender mismatch with adjective agreement) rule and fixture suite.

Purpose: Flag when a French adjective preceding a noun disagrees in gender with that noun. The existing es-fr-gender.js rule catches article-noun mismatches, but misses adjective-form errors like "un bon humeur" where "bon" (masculine) should be "bonne" (feminine) before "humeur" (f). This rule adds the adjective-gender agreement layer.
Output: New fr-adj-gender.js rule file with >= 30 positive + >= 15 acceptance fixtures.
</objective>

Create `extension/content/spell-rules/fr-adj-gender.js` using the IIFE self-registering pattern.

Rule shape:
- id: `'fr-adj-gender'`
- languages: `['fr']`
- priority: 16 (after es-fr-gender 15, before grammar 20)
- severity: `'warning'`

Inline data: a curated `FR_ADJ_FEM_IRREGULARS` map for common irregular feminine forms:
```
bon -> bonne, beau -> belle, vieux -> vieille, nouveau -> nouvelle,
fou -> folle, blanc -> blanche, sec -> seche, long -> longue,
gros -> grosse, faux -> fausse, doux -> douce, gentil -> gentille,
public -> publique, frais -> fraiche
```

And a `feminize(adj)` function that:
1. Checks FR_ADJ_FEM_IRREGULARS first
2. If adj ends in 'e', returns adj unchanged (already feminine-compatible)
3. Applies regular patterns: -er -> -ere, -eux -> -euse, -if -> -ive, -el/-en/-on -> double consonant + e
4. Default: adds 'e'

Also a `masculinize(adj)` reverse function or a `isMasculineForm(adj)` check. Since we're checking pre-nominal adjectives, we need to detect when a masculine adjective form precedes a feminine noun.

check(ctx) logic:
1. Only process FR language
2. Get `nounGenus` from `ctx.vocab.nounGenus`
3. For each token at index i:
   - Check if token i+1 exists and is a known noun (in nounGenus)
   - If yes, get the noun's gender from nounGenus
   - Check if token i is a known adjective: use `ctx.vocab.isAdjective` set OR check if the word has a known feminine form via feminize()
   - If the adjective is in masculine form and the noun is feminine: flag, suggest feminize(adj)
   - If the adjective is in feminine form and the noun is masculine: flag, suggest masculinize(adj)
4. Only check pre-nominal position (adjective immediately before noun). Post-nominal adjectives are structurally different and handled elsewhere.
5. Skip if the adjective is already gender-compatible with the noun.

Important: coordinate with es-fr-gender.js to avoid double-flagging. The existing rule flags article-noun mismatch. This rule flags adjective-noun mismatch. They target different tokens (article vs adjective), so dedupeOverlapping should handle it if both fire. If both fire on the same span, this rule defers (priority 16 > 15, lower priority wins in dedup).

Also handle the case where the adjective is between an article and a noun: `un [bon] humeur`. The existing es-fr-gender.js looks at prevPrev for article, so "un" -> article check against "humeur" is handled there. This rule separately checks "bon" against "humeur".

explain(finding):
- nb: "Adjektivet <em>{original}</em> har feil kjønn — bruk <em>{fix}</em> foran dette substantivet."
- nn: "Adjektivet <em>{original}</em> har feil kjønn — bruk <em>{fix}</em> framfor dette substantivet."
- severity: 'warning'

Create `fixtures/fr/adj-gender.jsonl` with >= 30 positive + >= 15 acceptance cases.

Positive cases: "un bon humeur", "un petit fille", "un grand maison", "le vieux femme", "un beau femme", "un nouveau voiture", "un blanc chemise", "un long histoire", "un gros erreur", "le faux monnaie", "un doux vie", "un gentil personne", "un bon idee", "un petit chambre", "un grand famille", "le bon humeur est important", "dans un bon humeur", and more sentence-context variations. Each should flag the adjective with the feminized suggestion.

Also include masculine noun + feminine adjective cases if applicable: "une bonne livre" -> flags "bonne", suggests "bon" (livre is masculine). This is rarer but students make both directions of error.

Acceptance cases (must NOT flag): "un bon livre", "une bonne humeur", "une belle femme", "le petit chat", "la petite maison", "la grande ville", "un nouveau livre", "une nouvelle voiture", "le vieux monsieur", "la vieille dame", "un beau jardin", "une belle fleur", "le long chemin", "la longue route", "un gros probleme" (probleme is masculine, so "gros" is correct).

Important: verify that "probleme" is masculine in nounGenus. If "La probleme" (article mismatch) is already caught by es-fr-gender.js, then fr-adj-gender would only add value when an adjective is also wrong. Test both scenarios in fixtures.
