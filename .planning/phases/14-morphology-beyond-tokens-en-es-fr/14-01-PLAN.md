  <name>Task 1: Build irregularForms index in vocab-seam-core.js and expose on seam</name>
  <files>extension/content/vocab-seam-core.js</files>
  <action>
Add a `buildIrregularForms(raw)` function in vocab-seam-core.js, placed near `buildParticipleToAux`. This function builds a Map keyed by wrong regular-pattern derivation forms, pointing to `{ correct, type, base }` objects.

For EN verbs with `verbClass: "irregular"`:
- Skip entries where `word` contains space or semicolon (multi-word phrasal verbs)
- Extract past tense from `conjugations.past.former.simple` and participle from `conjugations.perfect.former.participle` or `conjugations.perfect.participle`
- Generate wrong regular past: `word + "ed"` (and `word + "d"` for verbs ending in "e", like come -> comed)
- Also handle consonant doubling: for short verbs ending in consonant (run -> runned), generate `word + word.slice(-1) + "ed"`
- Map each wrong form to `{ correct: pastForm, type: "past", base: word }`
- Skip if the generated wrong form equals the actual correct form

For EN nouns with irregular plurals:
- Skip multi-word entries
- Get `forms.plural` (or `plural` field)
- Check if plural is NOT regular (not `word + "s"`, not `word + "es"`, not y->ies pattern)
- Generate wrong regular plural: `word + "s"` (and `word + "es"`)
- Map: `{ correct: plural, type: "plural", base: word }`

Call `buildIrregularForms(raw)` inside `buildIndexes` (after `buildParticipleToAux`), only for `lang === "en"`. Store result in state and expose as `getIrregularForms()` on the seam API (returns the Map, empty Map for non-EN). Build from raw data banks directly (not from wordList), ensuring feature-gate independence per Pitfall 3 from research.

Also expose `irregularForms` in the return object of `buildIndexes` so it's available via `vocab.irregularForms` in the spell-check ctx.
  </action>
