  <action>
  **A. benchmark-texts/expectations.json — initial manifest:**

  Create an expectations manifest that maps benchmark lines to expected rule firings. Structure:
  ```json
  {
    "_doc": "Machine-checkable benchmark expectations. Key = lang.line_number (1-based). Value = { rule_id, severity, priority_band }. check-benchmark-coverage validates each.",
    "entries": {}
  }
  ```

  Start with an empty `entries` object. Plan 03 will populate it when rules land. The gate should pass when entries is empty (nothing to check = all expectations met). This allows the gate to ship NOW and be populated incrementally.

  **B. scripts/check-benchmark-coverage.js:**

  Follow the existing gate pattern (check-explain-contract.js as template):

  1. Read `benchmark-texts/expectations.json`.
  2. If `entries` is empty, print `[check-benchmark-coverage] PASS: 0 expectations, nothing to check` and exit 0.
  3. For each entry in `entries`:
     - Parse the key as `{lang}.{line_number}` (1-based).
     - Read the corresponding `benchmark-texts/{lang}.txt` file.
     - Extract line `line_number` (1-based).
     - Load spell-check-core.js + all rule files (same loading pattern as check-fixtures.js).
     - Load vocab data for that language via vocab-seam-core.js buildIndexes.
     - Run `check(lineText, vocabIndexes, { lang })`.
     - Assert the expected `rule_id` appears in findings.
  4. Count passed/failed. Print summary. Exit 1 on any failure, exit 0 on all pass.

  Priority band reporting: group expectations by P1/P2/P3 (from `priority_band` field in manifest), print per-band flip-rate percentages. The gate itself fails on ANY missed expectation regardless of band — the percentage reporting is informational for phase-close decisions (roadmap says P1=100%, P2>=80%, P3>=50%).

  Important: The gate must handle the case where vocab data files don't exist for a language by skipping those expectations with a warning (not a failure), since not all languages may have data synced.

  **C. scripts/check-benchmark-coverage.test.js:**

  Paired self-test following check-explain-contract.test.js pattern:

  1. **Broken scenario:** Create a temporary expectations.json with a fake entry `"en.99999"` pointing to rule_id `"nonexistent-rule"`. Run the gate. Assert exit 1 (the line doesn't exist or rule doesn't fire).
  2. **Well-formed scenario:** Restore the real expectations.json (empty entries). Run the gate. Assert exit 0.
  3. Clean up temporary files.

  Use `child_process.execSync` with `stdio: 'pipe'` to capture output. Use try/catch for exit code detection.

  **A. scripts/check-governance-data.js:**

  Follow the check-spellcheck-features.js pattern (checks data shape in bundled vocab):

  1. Define EXPECTED_BANKS: a list of bank keys that governance rules depend on. Initially: `['registerbank', 'collocationbank', 'phrasebank']`. These will NOT exist in the data files until papertek-vocabulary lands them and sync-vocab runs. The gate should handle this gracefully.
  2. Design: The gate checks that for each language that HAS governance data, the expected bank keys are present and non-empty. Languages that don't have the banks yet are SKIPPED with a note (not a failure) — this allows incremental data landing.
  3. Additionally, when banks ARE present, validate structural shape:
     - registerbank entries should have `word` and `formal` fields.
     - collocationbank entries should have `trigger` and `fix` fields.
     - phrasebank entries should have `trigger` and `suggestion` fields.
  4. The gate passes when: (a) no governance banks found in any language (pre-data-sync state = PASS with "no governance data found" note), OR (b) all found banks pass shape validation.
  5. The gate fails when: any found bank has entries that fail shape validation.

  Print per-language summary. Exit 0 on pass, 1 on fail.

  **B. scripts/check-governance-data.test.js:**

  Paired self-test:

  2. **Well-formed scenario:** Create a temp file with valid bank shape. Assert gate passes.
  3. Or simpler: use the "no data found" pass-through (current state of repo has no governance banks). Assert the gate exits 0 today. Then create a temp data file with broken shape, assert exit 1.
  4. Clean up.
