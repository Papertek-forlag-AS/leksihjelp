  <name>Task 2: F36-3 + F36-4 fix (DE kein/keine paradigm)</name>
  <files>extension/content/spell-rules/de-gender.js, tests/fixtures/de/de-gender.json</files>
  <action>
The current `de-gender.js` ARTICLE_GENUS map handles definite (der/die/das) and one indefinite case (eine for feminine). The paradigm gap is that:
- `kein` is the masculine/neuter form of the negative indefinite article
- `keine` is the feminine (and plural) form

Mirror cases:
- `Ich habe kein Zeit` — `Zeit` is feminine; should be `keine Zeit`
- `Ich sehe keine Mann` — `Mann` is masculine; should be `kein Mann`

Implementation:

1. Extend `ARTICLE_GENUS` with kein/keine entries (treat parallel to ein/eine):
   ```js
   const ARTICLE_GENUS = {
     'der': 'm',
     'die': 'f',
     'das': 'n',
     'eine': 'f',
     'keine': 'f',  // F36-3/F36-4: negative indefinite, feminine + plural
   };
   ```
   And mirror the existing `ein` patch block (lines 111-123) for `kein`:
   ```js
   // F36-3: Manual check for 'kein' vs feminine nouns
   if (prev && prev.word === 'kein' && nounGenus.get(t.word) === 'f') {
     out.push({
       rule_id: 'gender',
       priority: rule.priority,
       start: prev.start,
       end: prev.end,
       original: prev.display,
       noun_display: t.display,
       actualGenus: 'f',
       fix: matchCase(prev.display, 'keine'),
       message: `Kjønn: "${prev.display} ${t.display}" skulle vært "keine ${t.display}"`,
     });
   }
   ```
   The `keine` → `kein` direction (F36-4) is handled by the main loop because `keine` is now in ARTICLE_GENUS as 'f' and a masculine noun follow-up will mismatch. Verify by tracing the loop logic for `keine Mann`: prev=`keine` (f), t=`Mann` (m), expected=f, actual=m, actual !== expected, correctArticle=`kein` (from GENUS_ARTICLE['m']) — fires correctly.

2. Confirm `Mann` and `Zeit` exist in `extension/data/de.json` nounbank with correct genus (Mann=m, Zeit=f). If either is missing, add inline (sync-vocab will pick them up next refresh) and document in 36-VERIFICATION.md.

3. Add fixture cases under `tests/fixtures/de/de-gender.json` (use `npm run add-fixture` or hand-edit consistent with existing schema):
   - `de-gender-pos-kein-fem` — `Ich habe kein Zeit.` expects rule `gender` on `kein` with fix `keine`
   - `de-gender-pos-keine-masc` — `Ich sehe keine Mann.` expects rule `gender` on `keine` with fix `kein`
   - `de-gender-neg-kein-masc` — `Ich habe kein Auto.` (Auto=n; some grammars accept kein for neuter; verify behaviour, document)
   - `de-gender-neg-keine-fem` — `Ich habe keine Zeit.` expects no finding

4. Run `npm run check-fixtures -- --rule gender` and full `npm run check-fixtures`. Both must pass at P=R=F1=1.000.
