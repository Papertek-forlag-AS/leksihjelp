The shipped failure mode is: user typed an FR verb, browser showed it flagged as a typo. Whatever the precise root cause (rule load? ctxLang stuck on NB? dedupe?), the user-visible symptom is `mangeait` getting flagged as a typo. The minimum-risk fix is to teach `nb-typo-fuzzy` to skip tokens that are recognised foreign-language conjugated forms even when ctx.lang doesn't match — this defuses the symptom regardless of which hypothesis is the true root cause, and is harmless when the rule SHOULD fire.

Rationale: nb-typo-fuzzy already runs for FR/DE/ES/EN (line 134) and already consults `vocab.verbForms` / `vocab.validWords`. The bug class is "ctx.lang is NB but user is typing FR" — in that state, FR-specific Maps in the seam may be empty (NB baseline doesn't populate them) AND validWords lacks the FR token. Adding a cross-language defensive check on the seam-exposed FR/DE/ES indexes ensures known foreign-language verb forms never get typo-flagged.

1. Edit `extension/content/spell-rules/nb-typo-fuzzy.js`. Inside the `check(ctx)` body, BEFORE the per-token typo emission, add a defensive guard:

   ```js
   // F36-1: Cross-language verb-form guard. When the seam-exposed
   // mood/aspect indexes recognise the token (in ANY language registered
   // through the seam), suppress the typo emission. Defends against
   // ctx.lang/vocab-state desync (vocab still on NB baseline while user
   // typed an FR token) AND against future lang-routing regressions.
   //
   // The check is conservative: only the *seam-surfaced* verb-form indexes
   // are consulted (frImparfaitToVerb, frPasseComposeParticiples,
   // frAuxPresensForms — and any future esPreterito*/dePreterit* indexes
   // gated by seam coverage). Empty Maps (the seam's safe default) make
   // this a no-op for languages that haven't hydrated.
   const FOREIGN_VERB_INDEX_KEYS = [
     'frImparfaitToVerb', 'frPasseComposeParticiples',
     // Add future cross-lang indexes here as they ship through the seam.
   ];
   const FOREIGN_VERB_SET_KEYS = ['frAuxPresensForms'];
   function tokenIsForeignVerbForm(lc) {
     for (const k of FOREIGN_VERB_INDEX_KEYS) {
       const m = ctx.vocab && ctx.vocab[k];
       if (m && typeof m.has === 'function' && m.has(lc)) return true;
     }
     for (const k of FOREIGN_VERB_SET_KEYS) {
       const s = ctx.vocab && ctx.vocab[k];
       if (s && typeof s.has === 'function' && s.has(lc)) return true;
     }
     return false;
   }
   ```

   Then in the per-token emission loop, after the existing `if (validWords.has(lc)) continue;` (or wherever the rule decides to emit), add:

   ```js
   if (tokenIsForeignVerbForm(lc)) continue;
   ```

   Place the call at the same gating point as `validWords.has(lc)` so the early-skip semantics are identical.
