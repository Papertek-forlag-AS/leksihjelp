  <name>Task 2: Implement check-synced-surface-version.js gate + paired self-test</name>
  <files>scripts/check-synced-surface-version.js, scripts/check-synced-surface-version.test.js</files>
  <action>
Mirror `scripts/check-popup-deps.js` skeleton line-for-line. Use the verbatim skeleton in RESEARCH.md "check-synced-surface-version.js skeleton" as starting point.

**`scripts/check-synced-surface-version.js` (the gate):**
- Strict CommonJS, zero deps, 'use strict'.
- ROOT, SYNCED_PATHS array as in interfaces block above.
- `git()` helper invoking `execSync('git ' + args, { cwd: ROOT, encoding: 'utf8' }).trim()`.
- main():
  1. Try `git describe --tags --abbrev=0`. On failure (no tags yet) → print informational PASS and exit 0.
  2. Run `git diff --name-only <tag>..HEAD -- <SYNCED_PATHS joined>`. If empty → print PASS with `no synced-surface changes since <tag>` and exit 0.
  3. Compare `JSON.parse(git('show <tag>:package.json')).version` vs `require(path.join(ROOT, 'package.json')).version`. If differ → PASS with bumped message + exit 0.
  4. Otherwise FAIL: stderr per-file diagnostic listing each changed file, plus two "fix:" lines: (a) `npm version patch/minor/major` + bump manifest.json + backend/public/index.html, (b) include `[lockdown-resync-needed]` in commit message. Exit 1.
- ANY git diff (modification, addition, deletion) inside synced paths counts (per CONTEXT lock). No whitespace exclusion.

**`scripts/check-synced-surface-version.test.js` (paired self-test):**
- Mirror `scripts/check-popup-deps.test.js` shape with rigorous signal-safe cleanup (per RESEARCH Pitfall 2).
- Plant target: a clearly-marked scratch file under one of the synced paths, e.g. `extension/i18n/__scratch-hyg05-test.json` (filename prefix `__scratch-` for visible-on-failure recovery).
- 3-step pattern:
  1. **Baseline**: run gate → must exit 0 (current state has zero synced-surface diff since v3.1, verified).
  2. **Plant bad**: write `__scratch-hyg05-test.json` with `{"_test": "hyg05"}`. The new file is now `git status` untracked. Note: the gate uses `git diff --name-only <tag>..HEAD` which only shows COMMITTED diffs since tag — untracked files won't trigger it. To plant a "synced-surface diff since tag" without committing, we must temporarily `git add` the file AND ALSO ensure the diff command sees it. Easier alternative: temporarily MODIFY an existing tracked file under a synced path (e.g. append a single trailing newline to `extension/i18n/<some-existing-file>`), run gate (must exit 1 mentioning the modified file), then restore via `git checkout -- <file>`. **Use this alternative approach** — modify an existing tracked synced-surface file; the diff is index-only; restoration via `git checkout` is signal-safe.
  3. **Plant good**: after restoration, baseline state holds → re-run gate, must exit 0.
- Cleanup harness signal-safe: `process.on('exit'/SIGINT/uncaughtException)` → spawnSync `git checkout -- <planted-path>` to guarantee restoration. Print "PASS"/exit 0 on success.
- Use try/finally + `let cleanedUp = false` flag to prevent double-cleanup races.

Add header comment identifying as HYG-05 release gate. Document the `extension/data/*.json` no-exclusion decision (per CONTEXT lock + RESEARCH Open Question 3) inline in the gate header — future maintainers will ask, answer it once.
