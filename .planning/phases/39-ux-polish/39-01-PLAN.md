# Plan: UX Polish & Contextual Pausing

This plan addresses a batch of UX feedback items from the `leksiscreenshots` folder. The primary architectural shift is moving from a single global "Pause" state (`lexiPaused`) to contextual, granular toggles (`predictionEnabled`, `spellCheckEnabled`, `widgetEnabled`) directly accessible where the features are used, with the popup Settings acting as the "mother of pauses".

## Scope
1. **Contextual Pausing:** Remove global `lexiPaused`. Wire local pause buttons (Word Prediction dropdown, Dictionary Widget, Aa Pill right-click) to disable their specific features and show a reactivation toast. Add `widgetEnabled` toggle to Settings.
2. **Skriv Button Visibility & Tooltip:** Fix the CSS/JS so the header pencil icon is strictly hidden while the inline placeholder is visible. Update tooltip to be clearer.
3. **Draggable Widget:** Add global drag-and-drop to the floating dictionary widget.
4. **First-Run Grammar Picker:** Add a step in the initial language download flow to choose the grammar level (Lite/Middels/Mye/Alt).

## Task 3: Global Draggable Position
<task_description>
The floating TTS widget and the inline lookup card (word-popup) are currently draggable, but their positions are not persisted globally across tabs. Update them to save and load their positions using `chrome.storage.local`.
</task_description>
<implementation_steps>
1. **`floating-widget.js`:**
   - In the drag `mouseup` handlers for both the TTS `widget` and the lookup `card`, save the coordinates to `chrome.storage.local` under `widgetPosition: { x, y }` and `lookupCardPosition: { x, y }` respectively.
   - On initialization/show, load these coordinates and apply them if they exist, ensuring the widget/card remains clamped within the visual viewport.
</implementation_steps>

## Task 4: First-Run Grammar Picker
<task_description>
Prompt the user for their preferred grammar level right after they download their first language.
</task_description>
<implementation_steps>
1. **`popup.html`:**
   - Add a step 2 container inside `#language-picker` (initially hidden) with the question "Hvor mye grammatikk vil du se?" and buttons for Lite, Middels, Alt.
2. **`popup.js`:**
   - When a language download succeeds in the onboarding flow, hide the language buttons and show the grammar buttons.
   - Clicking a grammar button saves `grammarLevel` and dismisses the onboarding.
   - If they click "Hopp over", it defaults to "Alt" (or leaves the existing default) and dismisses.
</implementation_steps>
