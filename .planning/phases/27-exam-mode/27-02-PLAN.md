**Part A — self-test (`scripts/check-exam-marker.test.js`):**

Mirror the structure of `scripts/check-explain-contract.test.js`:

1. Use `spawnSync('node', [GATE_PATH], { env, encoding: 'utf8' })` to run the gate.
2. Plant scratch rule files in `os.tmpdir()` (NOT in the repo). For each scenario, write a small JS file that pushes a rule to `host.__lexiSpellRules`.
3. Scenario 1 — malformed marker (gate must fire):
   - Scratch rule pushes `{ id: 'scratch-bad', exam: { safe: 'yes', reason: '' } }` (bad: safe is string, reason empty).
   - Spawn gate with `LEXI_EXAM_MARKER_EXTRA_TARGETS=<scratchPath>`.
   - Assert `result.status === 1`. If not, throw with `result.stderr + result.stdout`.
4. Scenario 2 — well-formed marker (gate must pass):
   - Scratch rule pushes `{ id: 'scratch-good', test: () => [], exam: { safe: true, reason: 'scratch-test', category: 'spellcheck' } }`.
   - Spawn gate with same env.
   - Assert `result.status === 0`.
5. Scenario 3 — missing marker entirely (gate must fire):
   - Scratch rule pushes `{ id: 'scratch-missing', test: () => [] }` (no exam field).
   - Assert `result.status === 1`.
6. Scenario 4 — invalid category (gate must fire):
   - `{ id: 'scratch-cat', test: () => [], exam: { safe: false, reason: 'x', category: 'made-up-category' } }`.
   - Assert `result.status === 1`.
7. Wrap everything in try/finally; cleanup scratch files in finally regardless of pass/fail.
8. On all-pass: `console.log('[OK] check-exam-marker:test — gate fires on malformed/missing/invalid-category, passes on well-formed');`
