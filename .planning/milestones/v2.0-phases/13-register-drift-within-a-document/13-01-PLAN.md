   c. Export both on the `tables` object alongside existing exports.
   d. Update the module header comment to mention Phase 13 document-drift tables.

Important: Do NOT use module-level mutable state. `detectDrift` must be a pure function.
Important: The existing single-pass loop will call `rule.check(ctx)` on document-rules too (they return `[]` — harmless no-op). The post-pass then calls `checkDocument`. This is by design.
