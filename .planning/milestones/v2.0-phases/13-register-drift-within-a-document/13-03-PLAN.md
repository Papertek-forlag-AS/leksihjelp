  <action>
Create `extension/content/spell-rules/doc-drift-nn-infinitive.js`:

1. Register as `kind: 'document'`, `id: 'doc-drift-nn-infinitive'`, `languages: ['nn']`, `priority: 204`, `severity: 'warning'`.
2. At IIFE init time (NOT per-call), build the infinitive classification map from NN vocab data:
   - Access `host.__lexiVocabStore?.nn` or load NN vocab via the seam if available.
   - IMPORTANT: The vocab data may not be available at IIFE init time in the browser (loaded async). Build the map lazily on first `checkDocument` call, then cache the map. This is acceptable — the map is derived deterministically from immutable vocab data, not mutable state.
   - Alternatively, build the map inside `checkDocument` each time (safe but slightly slower). Given typical student text sizes, this is fine.
   - Strategy: Iterate NN verbbank entries. For each entry with `infinitiv` as an array of 2+ forms:
     - Strip the `"a "` prefix from each form to get the bare infinitive.
     - If one form ends in `-a` and another ends in `-e`, record both: `akseptera -> 'a-infinitiv'`, `akseptere -> 'e-infinitiv'`.
   - Verbs with only one infinitive form: skip (register-neutral).
   - Store as a Map<string, string> where key is the bare infinitive and value is the register name.
3. `check(ctx)` returns `[]`.
4. `checkDocument(ctx, findings)`:
   a. Build or retrieve the infinitive classification map.
   b. Scan tokens: for each token, check if `tok.word` is in the map -> marker with the map's register value.
   c. Skip suppressed spans and build pass-1 flagged-span Set.
   d. Call `detectDrift(markers, 3)`.
   e. Build findings for minority markers.
   f. Fix suggestions: if minority is a-infinitiv, suggest the e-infinitiv counterpart (and vice versa). The map should store the counterpart form alongside the register classification.
5. `explain(finding)`:
   - NB: "Teksten blandar a-infinitiv og e-infinitiv. Mesteparten brukar [dominant]. Endre '[minority word]' til '[suggested form]'."
   - NN: "Teksta blandar a-infinitiv og e-infinitiv. Mesteparten nyttar [dominant]. Endre '[minority word]' til '[suggested form]'."

CRITICAL: The infinitive map must be built from VOCAB DATA, not from regex on word endings. Per RESEARCH.md Pitfall 4: `/-a$/` regex catches non-infinitive words. Only classify tokens that match known dual-form infinitives from the verbbank.

Data structure recommendation for the map: `Map<string, {register: string, counterpart: string}>` so `akseptera -> {register: 'a-infinitiv', counterpart: 'akseptere'}`.

Create fixture file `tests/fixtures/doc-drift-nn-infinitive.json`:

- 15+ acceptance fixtures covering: consistent a-infinitiv, consistent e-infinitiv, below-threshold, non-verb words ending in -a, verbs with single infinitive form only
- CRITICAL acceptance: "Eg likar boka mi." — "boka" is a noun, NOT an infinitive marker
  </action>
