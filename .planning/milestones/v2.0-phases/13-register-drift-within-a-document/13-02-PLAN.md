Create `extension/content/spell-rules/doc-drift-de-address.js` following the IIFE pattern from RESEARCH.md "Rule File Pattern":

1. Register as `kind: 'document'`, `id: 'doc-drift-de-address'`, `languages: ['de']`, `priority: 201`, `severity: 'warning'`.
2. `check(ctx)` returns `[]` (no-op).
3. `checkDocument(ctx, findings)`:
   a. Collect DE address markers per RESEARCH.md "DE du/Sie Marker Collection":
      - `du` (any case) -> informal
      - `Sie` mid-sentence (capital S, NOT sentence-start) -> formal
      - `Ihnen`, `Ihr`/`Ihre`/`Ihrem`/`Ihren` (capital I, mid-sentence) -> formal
      - `dein`/`deine`/`deinem`/`deinen`/`deiner` -> informal
      - `dir` -> informal, `dich` -> informal
      - 2sg verb forms ending in `-st` when in `ctx.vocab.knownPresens` -> informal
      - Skip lowercase `sie` (3rd person). Skip `Sie` at sentence start (ambiguous).
      - Skip tokens in `ctx.suppressedFor?.structural`.
   b. Build a Set of `start:end` keys from pass-1 `findings` to skip already-flagged spans.
   c. Call `detectDrift(markers, 3)` from grammar-tables.
   d. If null, return `[]`. Otherwise build findings for each minority marker not already flagged.
   e. Each finding: `rule_id: 'doc-drift-de-address'`, span from marker, `fix` suggesting dominant-register form, pedagogical `message` naming the dominant register.
4. `explain(finding)`:
   - Return `{ nb: '...', nn: '...', severity: 'warning' }` with student-friendly text per CONTEXT.md: name the dominant register and offer a concrete fix.
   - NB: "Du blander du-form og Sie-form i teksten. Mesteparten bruker [dominant]. Endre dette til [dominant]-form."
   - NN: Same pattern in nynorsk.

Create fixture file `tests/fixtures/doc-drift-de-address.json` with:
- 30+ positive fixtures (documents with du/Sie mixing, various ratios)
- 15+ acceptance fixtures (consistent register, below threshold, tied, quoted passages)
- Follow the >=2x acceptance-vs-positive ratio mandate

Marker identification for `Sie` at sentence start: check if `tok.start` equals a `ctx.sentences[n].start` (after trimming leading whitespace). If so, skip — it's ambiguous.
