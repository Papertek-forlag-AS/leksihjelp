**Create extension/content/spell-rules/nb-v2.js:**

IIFE rule following the standard pattern. Key design:
- `id: 'nb-v2'`, `languages: ['nb', 'nn']`, `priority: 65`, `severity: 'warning'`
- Uses `tokensInSentence`, `findFiniteVerb`, `findSubordinator`, `isMainClause` from `__lexiSpellCore`
- Also define WH_WORDS as `new Set(['hvorfor', 'hvordan', 'hva', 'hvem', 'hvilken', 'hvilke', 'hvor', 'når'])` for NB and `new Set(['kvifor', 'korleis', 'kva', 'kven', 'kvar', 'når'])` for NN
- Honor `ctx.suppressedFor.structural` (skip suppressed tokens)
- Detection algorithm per Research:
  1. For each sentence, get token range via `tokensInSentence`
  2. Skip sentences with fewer than 3 tokens
  3. Find finite verb; skip if none
  4. Check isMainClause — if NOT main clause (subordinator before verb), skip
  5. If verb is at relative position 0 or 1 (V1 or V2), skip (correct order)
  6. If relative position >= 2: scan positions between sentence start+1 and verb for a subject pronoun
  7. If found: flag the span from subject to verb with suggestion showing inverted order (verb before subject)
  8. Special wh-word handling: if token at position 0 is a wh-word and position 1 is a subject pronoun and position 2+ is verb → flag
  9. Embedded wh-clause guard (Pitfall 6): if a finite verb exists BEFORE the wh-word in the sentence, the wh-clause is embedded — do NOT flag
- `explain`: NB = `"I norske hovedsetninger skal verbet stå på plass 2..."`, NN = `"I norske hovudsetningar skal verbet stå på plass 2..."`

**Create extension/content/spell-rules/de-v2.js:**

Nearly identical logic to nb-v2 but with DE-specific sets:

- DE wh-words: warum/wie/was/wer/welcher/welche/welches/wo/wann/woher/wohin
- Same V2 detection algorithm
- `explain`: NB = `"I tyske hovedsetninger skal verbet stå på plass 2..."`, NN = `"I tyske hovudsetningar skal verbet stå på plass 2..."`
