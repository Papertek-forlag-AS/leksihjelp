  - IIFE with standard registration pattern.

  - Read `SEIN_VERBS` and `BOTH_AUX_VERBS` from `host.__lexiGrammarTables`.
  - Read `participleToAux` Map from `ctx.vocab` (via `ctx.vocab.getParticipleToAux()` or `ctx.vocab.participleToAux`).
  - Detection algorithm per sentence:
    1. Look for auxiliary forms of `haben` or `sein` — conjugated forms: `habe/hast/hat/haben/habt/hatte/hattest/hatten/hattet` (haben), `bin/bist/ist/sind/seid/war/warst/waren/wart` (sein).
    2. After finding an auxiliary, scan forward (up to 5 tokens) for a past participle (starts with `ge-` or matches a known participle in `participleToAux`).
    3. Look up the participle in `participleToAux`. If it maps to a specific auxiliary and the found auxiliary is WRONG → flag.
    4. If the participle maps to `'both'` → skip (no flag).
    5. Suggest the correct auxiliary form (matching conjugation: if student wrote "habe" → suggest "bin"; if "hat" → suggest "ist"; etc.).

  **2. Create `extension/content/spell-rules/de-compound-gender.js`:**
  - IIFE with standard registration pattern.
  - `id: 'de-compound-gender'`, `languages: ['de']`, `priority: 71`.
  - Detection algorithm per sentence:
    1. For each token, check if it's preceded by a German article (der/die/das/ein/eine).
    2. If the token (capitalized form) is NOT in `vocab.nounGenus` → candidate for compound inference.
    3. Apply greedy longest-suffix split: for i = 1 to word.length - 3, check if `word.slice(i).toLowerCase()` is in `vocab.nounGenus`. Take the FIRST match (longest suffix).
    4. If a suffix matches: its gender is the compound's gender. Compare with the article's expected gender (nominative context — use DEF_ARTICLE_CASE/INDEF_ARTICLE_CASE from grammar-tables).
    5. If the article gender doesn't match the inferred gender → flag with correct article.
    6. Be careful with case: the compound word as typed may be capitalized (German nouns). The nounGenus lookup should use lowercase.
    7. Handle linking elements: German compounds sometimes insert `-s-`, `-n-`, `-en-`, `-er-` between components. Try stripping common linking elements when the direct suffix lookup fails. Example: "Schultasche" = "Schul" + "tasche" (no linker), "Arbeitszimmer" = "Arbeits" + "zimmer" (s-linker). After removing the linker, retry the suffix lookup.
  - Explain: student-friendly text about compound-noun gender (last component determines gender).
  - `severity: 'warning'` (P2).
