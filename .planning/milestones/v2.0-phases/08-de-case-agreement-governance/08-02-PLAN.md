  - ACCEPTANCE: "für den Mann" → no flag (correct acc masculine)
  - ACCEPTANCE: "aus einer Flasche" → no flag (correct dative feminine indef)
  - Two-way preps (in/auf/an etc) flag at warn severity only (not error) since motion/location ambiguity
  - Adjacent-article scope only: prep [adj]? article [adj]? noun — max 3 tokens after preposition

  - ACCEPTANCE: "dass ich aufstehe" → no flag (subordinate clause — unsplit is CORRECT)
  - ACCEPTANCE: "ich stehe um sieben auf" → no flag (correctly split)
  - ACCEPTANCE: "wenn er einschläft" → no flag (subordinate)
  - Must check isMainClause() before flagging
  - Must verify the prefix doesn't appear separately later in same clause

  **1. Create `extension/content/spell-rules/de-prep-case.js`:**
  - IIFE with `host.__lexiSpellRules` registration pattern.
  - `id: 'de-prep-case'`, `languages: ['de']`, `priority: 68` (structural tier, same band as word-order rules).
  - Read `PREP_CASE`, `DEF_ARTICLE_CASE`, `INDEF_ARTICLE_CASE` from `host.__lexiGrammarTables`.
  - Detection algorithm per sentence:
    1. Iterate tokens. When a token's lowercase matches a key in `PREP_CASE`, note the required case.
    2. Scan next 1-3 tokens for an article form (definite or indefinite).
    3. Look up the article form's possible cases from the article-case tables.
    4. If the required case (from prep) has ZERO overlap with the article's possible cases → flag.
    5. For two-way preps (`acc/dat`): only flag if the article's possible cases include NEITHER `akkusativ` NOR `dativ` (e.g., genitive article after a two-way prep).
    6. For genitive preps (`wegen/statt/trotz/während`): flag at warn severity (colloquial dative is widespread).
    7. Check `ctx.suppressedFor.structural` before emitting.
  - Also check the noun's gender against the article's gender-in-required-case: if `vocab.nounGenus` has the noun, verify the article gender matches the noun gender in the required case. This catches "mit den Schule" where "den" is m.acc but "Schule" is f and the correct dative feminine article is "der".
  - Explain: `explain: (f) => ({ nb: '...', nn: '...', severity: 'warning' })` — student-friendly text explaining which case the preposition requires.
  - Suggestion: the correct article form for the noun's gender in the required case.

  **2. Create `extension/content/spell-rules/de-separable-verb.js`:**
  - IIFE with standard registration pattern.
  - `id: 'de-separable-verb'`, `languages: ['de']`, `priority: 69`.
  - Read `SEPARABLE_PREFIXES` from `host.__lexiGrammarTables`.
  - Detection algorithm per sentence:
    1. For each token, check if it matches a known separable verb's unsplit form (prefix + finite stem). Use the `_finiteStems` approach from `de-v2.js` or check if stripping a prefix from `SEPARABLE_PREFIXES` yields a form in `vocab.knownPresens` or `vocab.knownPreteritum`.
    2. Verify the token is in a main clause using `isMainClause(ctx, sentStart, sentEnd)`.
    3. Verify the prefix does NOT appear as a separate particle later in the clause (scan remaining tokens in sentence for the prefix).
    4. If all conditions met → flag with suggestion to split.
  - Explain: student-friendly text about separable verbs in main clauses.
  - `severity: 'warning'` (P2).

  **3. Create fixture files** with ≥30 positive + ≥15 acceptance cases each:

  - `fixtures/de/separable-verb.jsonl`: Cover common separable verbs (aufstehen, anfangen, einkaufen, anrufen, etc.), main vs subordinate clause distinction, correctly-split forms, prefix appearing separately.
