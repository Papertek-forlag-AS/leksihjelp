**Rule file (es-subjuntivo.js):**

Create following the IIFE pattern from es-ser-estar.js / es-por-para.js. Key properties:
- `id: 'es-subjuntivo'`
- `languages: ['es']`
- `priority: 60` (between ser-estar at 50 and modal-verb at 110)
- `severity: 'warning'`

**Detection algorithm:**
1. Iterate sentences via `ctx.sentences`. For each sentence, get tokens via `tokensInSentence(ctx, si)`.
2. Walk tokens looking for multi-word trigger matches against `ES_SUBJUNTIVO_TRIGGERS` from grammar-tables.js. To match multi-word triggers: at each token position, concatenate 2-5 consecutive tokens (lowercased, space-joined) and check if any prefix matches a trigger in the Set. This must end with "que".
3. After matching a trigger ending in "que", scan forward past optional subject pronouns (yo, tu, el, ella, usted, nosotros, vosotros, ellos, ellas, ustedes -- and also articles/adjectives like "mi", "su", "un", "una" + noun for NP subjects like "mi hermano").
4. Find the next token that is a known verb form. Check via `ctx.indexes.esPresensToVerb` -- if the token (lowercased, and accent-stripped variant) is in this Map, it's an indicative present form.
5. Look up `{ inf, person }` from `esPresensToVerb`. Then look up `esSubjuntivoForms.get(inf + '|' + person)` to get the subjunctive form.
6. If the subjunctive form exists AND differs from the written form: create a finding with `original` (written form), `fix` (subjunctive form), `trigger` (the matched trigger phrase).

**Person mapping strategy (from research):** Use the person of the indicative form itself. If `esPresensToVerb.get('viene')` returns `{ inf: 'venir', person: 'el/ella/usted' }`, suggest `esSubjuntivoForms.get('venir|el/ella/usted')` = "venga". This avoids the NP-person-detection problem entirely.

**Accent stripping:** Apply `token.normalize('NFD').replace(/[̀-ͯ]/g, '')` for lookup, matching the pattern from fr-etre-avoir.js. Students often omit tildes.

**CRITICAL guard against relative-clause false positives:** The trigger must be a COMPLETE multi-word phrase from the Set. "El libro que viene" does NOT match because "libro que" is not in the trigger set. The trigger-start word must be the verb/expression itself (quiero, espero, etc.), not a noun.

**explain function:** Return `{ nb, nn, severity: 'warning' }`. The nb/nn strings should explain that after the trigger phrase, subjunctive mood is required, showing the correct form. Use `escapeHtml` on interpolated values.

**Fixture file (es-subjuntivo.jsonl):**

Create >=30 positive fixtures and >=15 acceptance fixtures (>=2x acceptance ratio).

Positive fixtures (should flag):
- "Quiero que mi hermano viene conmigo" (viene -> venga)
- "Espero que tu hablas con ella" (hablas -> hables)
- "Dudo que el tiene razon" (tiene -> tenga)
- "Es importante que nosotros comemos bien" (comemos -> comamos)
- "No creo que ella sabe la respuesta" (sabe -> sepa)
- "Es posible que llueve manana" (llueve -> llueva)
- "Prefiero que tu vienes temprano" (vienes -> vengas)
- Various trigger phrases covering the full set
- Cases with and without accents on student input
- Cases with NP subjects between que and verb ("mi hermano", "el profesor")

Acceptance fixtures (should NOT flag):
- "Creo que mi hermano viene manana" (creo que = indicative, NOT in trigger set)
- "Dice que habla espanol" (dice que = indicative)
- "El libro que viene de la biblioteca" (relative clause)
- "La persona que tiene el libro" (relative clause)
- "Quiero ir al cine" (no "que" clause)
- "Es verdad que tiene razon" (es verdad que = indicative)
- Sentences with correct subjunctive forms already used

Follow JSONL fixture format used by existing rules (one JSON object per line with `text`, `lang`, `expected` fields).

**Rule file (es-imperfecto-hint.js):**

Create following the same IIFE pattern. Key properties:
- `id: 'es-imperfecto-hint'`
- `languages: ['es']`
- `priority: 65` (near subjuntivo)
- `severity: 'hint'` -- CRITICAL: this MUST be hint, never warning or error. The success criteria explicitly require hint-tier rendering.

**Detection algorithm:**
1. Iterate sentences. For each sentence, get tokens.
2. Walk tokens looking for aspectual adverb matches:
   - Single-word: check token (lowercased, accent-stripped) against `ES_PRETERITO_ADVERBS` or `ES_IMPERFECTO_ADVERBS`
   - Multi-word: at each position, concatenate 2-4 consecutive tokens and check against `ES_PRETERITO_PHRASES` or `ES_IMPERFECTO_PHRASES`
3. When a preterito adverb is found (e.g., "ayer"), scan the sentence for verb forms. Check each token against `ctx.indexes.esImperfectoForms` reverse-lookup -- wait, we need the inverse: we need to detect if a verb is imperfecto when preterito is expected. So:
   - If preterito adverb found: scan for imperfecto verb forms. A token is imperfecto if it appears in a reverse map built from imperfecto data. But we may not have a direct "imperfecto form -> infinitive" map. **Alternative approach:** check if the token is in `esPreteritumToVerb` (it's NOT preterito, so check the inverse). Actually, the simpler approach: scan tokens for verbs that are in the imperfecto form set. We need to check: is this token an imperfecto form? We can do this by checking if any `esImperfectoForms` value matches the token. But that's O(n). **Better:** Build an `esImperfectoToVerb` map (imperfecto form -> {inf, person}) in Plan 01's vocab-seam extension, alongside `esPreteritumToVerb`. If Plan 01 did not build this, the rule should build a lazy reverse map on first call from `esImperfectoForms`.
   - If imperfecto adverb found (e.g., "siempre"): scan for preterito verb forms using `esPreteritumToVerb`.
4. When mismatch found: create a finding with the adverb as trigger, the mismatched verb, and suggestion of the correct aspect form (look up the infinitive+person in the opposite aspect's form map).

**Lazy reverse-map fallback:** If `esImperfectoToVerb` is not available from indexes, build it lazily on first `check()` call by inverting `esImperfectoForms` (which maps `inf|person` -> form, so invert to form -> `{inf, person}`).

**"mientras" guard (from research):** Only match bare "mientras" (not "mientras que") for imperfecto hint. "Mientras que" is contrastive, not temporal.

**explain function:** Return `{ nb, nn, severity: 'hint' }`. Educational tone: "Tidsuttrykket X antyder at verbet burde vaere i [preterito/imperfecto]-form" (Norwegian explain for the student).

**Fixture file (es-imperfecto-hint.jsonl):**

>=30 positive + >=15 acceptance fixtures.

Positive:
- "Ayer yo caminaba por el parque" (imperfecto with preterito adverb -> suggest preterito)
- "Siempre yo comi en ese restaurante" (preterito with imperfecto adverb -> suggest imperfecto)
- "Anoche ella hablaba con su madre" (imperfecto + preterito adverb)
- "Cada dia yo fui a la escuela" (preterito + imperfecto adverb)
- "Normalmente el comio temprano" (preterito + imperfecto adverb)
- Multi-word adverb phrases: "la semana pasada", "todos los dias", "de vez en cuando"

Acceptance:
- "Ayer yo camine al parque" (preterito matches preterito adverb -- correct)
- "Siempre caminaba por aqui" (imperfecto matches imperfecto adverb -- correct)
- "Mientras ella hablaba, yo escuchaba" (imperfecto with mientras -- correct)
- "Mientras que es dificil, lo intento" (mientras que = contrastive, no flag)
- Sentences with no temporal adverbs at all
- Ambiguous cases where either aspect could be correct
