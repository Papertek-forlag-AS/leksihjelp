**fr-pp-agreement.js:** Create as IIFE. Key implementation details:
- id: 'fr-pp-agreement', languages: ['fr'], priority: 72, severity: 'hint'
- Lazy-init grammar tables at first check() call (same pattern as de-prep-case)
- Feature gate: `if (!ctx.vocab || !ctx.vocab.isFeatureEnabled || !ctx.vocab.isFeatureEnabled('grammar_fr_pp_agreement')) return [];`
- Rule header comment must document 10.3b deferred scope:
  ```
  // 10.3b deferred to v3.0:
  //   - l' DO-pronoun (requires antecedent gender resolution)
  //   - que relative pronoun (distance > adjacent window)
  //   - Pronominal verbs with reflexive DO
  //   - Elided DO requiring gender inference
  ```
- Detection algorithm:
  1. Iterate tokens. If token is `la` or `les`:
  2. Check if next token is a known avoir form (from FR_AVOIR_FORMS)
  3. Check if token after that is a known participle (from ctx.vocab.participleToAux where aux !== 'etre', or just any word that looks like a PP — simpler: check participleToAux Map)
  4. If `la` + avoir + PP: check if PP ends with -e (feminine agreement). If not, flag PP with suggestion adding -e.
  5. If `les` + avoir + PP: check if PP ends with -s (plural agreement). If not, flag PP with suggestion adding -s.
  6. Skip if PP already has correct ending.
  7. Note: The "la" here is the DO pronoun, not the article. In practice both trigger the same check since the adjacent-window pattern [la][avoir-form][PP] is the same regardless.

- For suggestion generation: simple suffix addition:
  - Feminine: add -e if not already ending in -e (mange -> mangee, pris -> prise, mis -> mise, fait -> faite)
  - Plural: add -s if not already ending in -s (mange -> manges, prise -> prises)
  - For `les` (plural), if PP doesn't end in -e and la context is feminine too, need -es. But since `les` can be masc or fem plural, just suggest adding -s for simplicity. This is the 10.3a narrow scope.

- explain returns {nb, nn} with student-friendly message about PP agreement with preceding DO
