NB entry schema (extension/data/nb.json):
```json
{

    "aktuell_adj": {

      "word": "på",

      "linkedTo": { "en": { "primary": "at_contr" } }
    }
  }
}
```

Target-language entries (e.g., de.json) do NOT have falseFriends or senses fields.

The reverse mapping: NB entry.linkedTo[currentLang].primary → target entry ID (e.g., "aktuell_adj")
This allows building: Map<targetEntryId, nbEntry> to enrich target results with NB data.

popup.js key variables:
- `allWords` — flattened from target dictionary (de/es/fr/en). Each entry spread from bank, with _bank and partOfSpeech added.
- `noWords` — flattened from NB/NN dictionary (loaded in loadDictionary).
- `noDictionary` — raw NB/NN dictionary object (banks intact).
- `currentLang` — target language code ('de', 'es', 'fr', 'en').
- `flattenBanks(dict)` — at line 539: iterates banks, spreads entries, does NOT preserve the bank key (wordId) on the entry.

popup.js search flow (performSearch at line 1048):
1. Searches `allWords` (target dict) for direct matches
2. Searches `noWords` (NB dict) for no→target two-way lookup via linkedTo
3. Both paths push { entry: targetFlatEntry, inflectionHint } into results
4. renderResults (line 1291) calls renderFalseFriends(entry) and renderSenses(entry) — these check entry.falseFriends/senses which are ALWAYS empty on target entries

floating-widget.js search flow (showInlineLookup at line 1025):
1. Loads ONLY the target-language dictionary
2. searchDictBanks returns first match from target dict
3. Renders falseFriends/senses from match — ALWAYS empty since target entries lack these fields

Modify `searchDictBanks` (line 1041-1061) to also track the entry key. Change the inner loop to iterate `Object.entries` instead of `Object.values`:
```js
function searchDictBanks(dictData, query) {
  const banks = Object.keys(BANK_TO_POS_KEY);
  for (const bank of banks) {
    const bankData = dictData[bank];
    if (!bankData || typeof bankData !== 'object') continue;
    for (const [entryId, entry] of Object.entries(bankData)) {
      if (!entry.word) continue;
      if (entry.word.toLowerCase() === query ||
          (getTranslation(entry) || '').toLowerCase() === query) {
        return {
          ...entry,
          _wordId: entryId,
          translation: getTranslation(entry),
          partOfSpeech: bankToPos(bank),
          gender: entry.genus ? genusToGender(entry.genus) : null,
          grammar: entry.explanation?._description || null,
          examples: entry.examples || []
        };
      }
    }
  }
  return null;
}
```

Then, after the match is found (after line 1080), enrich it with NB data:
```js
// Enrich match with NB falseFriends/senses
if (match && nbDict && match._wordId) {
  for (const bank of Object.keys(nbDict)) {
    const bankData = nbDict[bank];
    if (!bankData || typeof bankData !== 'object') continue;
    for (const [, nbEntry] of Object.entries(bankData)) {
      if (!nbEntry.linkedTo?.[currentLang]?.primary) continue;
      if (nbEntry.linkedTo[currentLang].primary !== match._wordId) continue;
      if (nbEntry.falseFriends) {

      if (nbEntry.senses) {

      }
    }
  }
}
```
