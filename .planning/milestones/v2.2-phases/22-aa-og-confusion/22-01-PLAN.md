<objective>
Implement a dedicated å/og confusion detection rule for Norwegian (NB/NN) spell-check.

Purpose: å/og is the single most common writing error for Norwegian students. The existing `nb-homophones.js` has basic å/og heuristics but lacks posture-verb exception handling and has limited trigger coverage. This plan creates a robust dedicated rule and removes the å/og logic from homophone to avoid double-flagging.

Output: Working `nb-aa-og.js` rule file with fixtures, explain contract, CSS wiring, and all release gates passing.
</objective>

       **Direction 1: "og" should be "å" (AAOG-01)**
       Check: token is "og", previous token is an INFINITIVE_TRIGGER (verb/adjective that takes infinitive complement), next token is a verb form (in verbForms or validWords has "å {next}").
       INFINITIVE_TRIGGERS: Expand the existing set from nb-homophones.js with more verbs:

       Plus adjective/adverb triggers: `godt, fint, viktig, vanskelig, lett, moro, gøy, kjedelig, umulig, klart, farlig, trygt, morsomt, enkelt, nødvendig, mulig, greit, dumt, lurt, rart, vanlig`

       **Direction 2: "å" should be "og" (AAOG-02)**
       Check: token is "å", next token is NOT a verb form (i.e., next is a noun, article, pronoun, preposition, adjective, or another non-verb word). Use the same heuristic as nb-homophones: check if next is in ARTICLES_POSSESSIVES, PRONOUNS, or common prepositions. Also check: if prev is a noun/adjective (not a verb trigger) AND next is a noun/adjective — coordinate structure that needs "og".

       **Posture verb exception (AAOG-03)**
       POSTURE_VERBS set (both NB and NN present + preteritum forms):
       `sitter, satt, sat, står, stod, sto, staar, ligger, lå, la, laag, går, gikk, gjekk, henger, hang, hengte, satt, stod, lå, gikk, sit, stod, låg, gjekk, hang`
       When prev token (or prev-prev if subject pronoun intervenes) is a posture verb form AND next token is a verb form → skip flagging. This is grammatically correct Norwegian ("sitter og leser" = "is sitting reading").

       **Additional safe patterns (no flag):**
       - "å" at sentence start followed by verb → valid infinitive marker
       - "og" between two parallel non-verb words → valid conjunction
       - Cursor proximity skip (standard pattern)
       - Suppressed token skip (standard pattern)
