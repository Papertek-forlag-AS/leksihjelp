  ```javascript
  function buildIndexes({ raw, bigrams, lang, isFeatureEnabled }) {
    const iff = isFeatureEnabled || (() => true);  // default: emit all forms (Node/test use)
    const wordList = buildWordList(raw, lang, iff);
    const { nounGenus, verbInfinitive, validWords, typoFix, compoundNouns } =
      buildLookupIndexes(wordList, lang);
    const normBigrams = bigrams ? normalizeBigrams(bigrams) : null;
    return {
      wordList,
      nounGenus, verbInfinitive, validWords, typoFix, compoundNouns,
      bigrams: normBigrams,
      freq: new Map(),          // empty in Phase 1; Phase 2 DATA-01 populates
      typoBank: typoFix,        // alias per CONTEXT rich surface
    };
  }
  ```

Step 3 — `init()` sequence:
  - Read `chrome.storage.local.get(['language', 'lexiPaused'])`.
  - Fall back to `currentLang = 'en'` and `paused = false` on absence.
  - Call `loadForLanguage(currentLang)`.
  - Register `chrome.runtime.onMessage.addListener(onMessage)`.

Step 4 — `async function loadForLanguage(lang)`:
  - Set `ready = false`.
  - Load raw vocab: try `window.__lexiVocabStore.getCachedLanguage(lang)` first (IndexedDB cache). On null/throw, fall back to `fetch(chrome.runtime.getURL('data/' + lang + '.json'))` + `res.json()`. Bundled languages: `['nb','nn','en']` — for others, skip the fetch and return early with an empty state (state = null, ready stays false). See CONTEXT "vocab-seam.js owns vocabulary loading end-to-end".
  - Load bigrams: `fetch(chrome.runtime.getURL('data/bigrams-' + lang + '.json'))`. If `!res.ok`, set bigrams to `null` (not an error — many languages have no bigrams file yet).
  - Read grammar-features toggle from storage to pass to the core as `isFeatureEnabled` (mirrors word-prediction.js pattern). Helper: read `chrome.storage.local.get('enabledGrammarFeatures')`; build a predicate `(id) => enabledSet.has(id)` with a sane default when the stored value is missing (treat as "all enabled" to avoid over-filtering at seam level — consumers filter further).
  - Call `state = core.buildIndexes({ raw, bigrams, lang, isFeatureEnabled })`.
  - Set `ready = true`; drain `readyCallbacks.splice(0)` safely with a try/catch around each call (matches the existing `onReady` pattern).

Step 5 — `onMessage` listener:
  - `LANGUAGE_CHANGED` → set `currentLang = msg.language`, call `loadForLanguage(currentLang)`.
  - `LEXI_PAUSED` → set `paused = !!msg.paused`.
  - `GRAMMAR_FEATURES_CHANGED` → call `loadForLanguage(currentLang)` (rebuild indexes with new toggles) per RESEARCH open-question #1.
  - Ignore `PREDICTION_TOGGLED` and `SPELL_CHECK_TOGGLED` — these are consumer-local policy flags per RESEARCH open-question #3.

Step 6 — Public API on `self.__lexiVocab`:
  Implement exactly the surface in the `<interfaces>` block. Critical detail: `onReady(cb)` must handle late subscribers:
  ```javascript
  onReady(cb) {
    if (ready) { try { cb(); } catch (_) {} return; }
    readyCallbacks.push(cb);
  }
  ```
  See pitfall #1 — without this, subscribers that arrive after `ready = true` are silently dropped.

  `getFrequency(word)`:

  ```javascript
  getFrequency: (word) => {
    if (!state || !state.freq) return null;
    const v = state.freq.get(word.toLowerCase());

  ```
  Phase 1 freq is empty → always returns null. That's the intended Phase-1 behavior; Phase 2 populates the data without changing the signature.

  `isTextInput(el)`: copy verbatim from `word-prediction.js:1829–1844` (currently inside the `__lexiPrediction` seam).

Step 7 — Initialization guard:
  Put `init()` at the bottom of the IIFE (inside the wrapper, before the `}())`) and call it once. Do NOT call on `DOMContentLoaded` — content scripts at `document_idle` already fire late enough.

Avoid:
- Do NOT expose a `self.__lexiPrediction` alias. CONTEXT explicitly rejects this ("deleted outright in the same commit — no alias, no adapter").
- Do NOT couple to spell-check or word-prediction state (no toggle listeners). Seam is consumer-agnostic.
- Do NOT call `fetch` inside the core — all I/O lives in this wrapper.
- Do NOT rely on `__lexiPrediction` being present. This file must initialize without it.
