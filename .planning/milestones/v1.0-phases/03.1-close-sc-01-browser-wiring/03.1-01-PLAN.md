<research_pitfalls>
Hot-take summary of 03.1-RESEARCH.md pitfalls that directly gate task implementation:

- **Pitfall 1 (Empty-Map-default):** Match `getValidWords()` / `getNounGenus()` style (return `new Map()` when state is not ready), NOT `getBigrams()` style (returns null). The rule's null-guard is `if (vocab && vocab.freq)` — an empty Map is truthy, so the guard passes and `vocab.freq.get(cand)` returns undefined; the `if (typeof z === 'number')` below catches that. No Zipf boost, no crash. Correct behavior.
- **Pitfall 2 (fixture suite does not exercise the adapter):** The fixture runner bypasses spell-check.js entirely, so a green fixture suite cannot prove the adapter wiring is correct. The contract assertion is the coverage; ship it.
- **Pitfall 6 (language-agnostic assertion):** Keep the contract assertion STATIC (regex on source file), not dynamic (runtime check on `vocab.freq.size`). A dynamic check would over-fire on DE/ES/FR/EN where no freq sidecar ships.
