  <action>
Create `nb-demonstrative-gender.js` following the IIFE pattern from nb-gender.js.

Rule specification:
- **id:** `nb-demonstrative-gender`
- **languages:** `['nb', 'nn']`
- **priority:** 12 (between nb-gender at 10 and other grammar rules; confirmed no existing rule uses 12)
- **severity:** `error`
- **Trigger words:** `den` (m/f), `det` (n), `denne` (m/f), `dette` (n) — with gender mapping per language
- **Fix mapping:** `{ m: 'den', f: 'den', n: 'det' }` (NB), same for NN; `denne/dette` maps similarly: `{ m: 'denne', f: 'denne', n: 'dette' }`
- **Logic:** For each token, check if it's a demonstrative. Look at next token (i+1) and 2-ahead (i+2, for adjective gap like "det store boka"). If the following/2-ahead token is a known noun in `nounGenus`, check gender match. If mismatch, emit finding with corrected demonstrative.
- **NB common-gender tolerance:** `den` accepts both m and f (same as nb-gender.js line 79-80: `lang === 'nb' && actual === 'f' && articleTok.word === 'en'`). Apply same pattern: `lang === 'nb' && actual === 'f' && demonstrative is 'den' or 'denne'`.
- **explain:** Three-beat copy with gender label, same pattern as nb-gender.js (use GENUS_TO_LABEL_KEY + getString).
- **Cursor exclusion:** Skip if cursor is within the noun token range (standard pattern).

Create fixtures in `fixtures/nb/nb-demonstrative-gender.jsonl`:
- Positive (should flag): "Det boka" (det+f noun), "Den huset" (den+n noun), "Dette boka" (dette+f noun), "Det store boka" (adjective gap), "Denne huset" (denne+n)
- Negative (should NOT flag): "Den boka" (correct), "Det huset" (correct), "Den store boka" (correct with adjective), "Det er fint" (expletive — "er" is not in nounGenus), "Det barnet" (correct n+n)

Do NOT modify nb-gender.js — the trigger word sets are disjoint so no collision avoidance needed.

<task type="auto">
  <name>Task 2: Create nb-triple-letter rule with fixtures</name>
  <files>extension/content/spell-rules/nb-triple-letter.js, fixtures/nb/nb-triple-letter.jsonl</files>
  <action>
Create `nb-triple-letter.js` following the IIFE pattern. This MUST be a separate rule file (not a modification to nb-typo-fuzzy — per STATE.md Pitfall 8).

Rule specification:
- **id:** `nb-triple-letter`
- **languages:** `['nb', 'nn']`
- **priority:** 45 (between typo-curated at 40 and typo-fuzzy at 50)
- **severity:** `error`
- **Logic:** For each token:
  1. Skip if `suppressed.has(i)` or cursor is within token range
  2. Skip if `validWords.has(t.word)` — known valid word, not a typo
  3. Test against regex `/(.)\\1\\1+/` to detect triple-or-more repeated chars
  4. If match: collapse ALL triple+ runs to double: `word.replace(/(.)\\1\\1+/g, '$1$1')`
  5. If collapsed form is in `validWords`, emit finding with fix = collapsed form (use `matchCase`)
  6. If collapsed form NOT in validWords, skip (might be a different kind of error)
- **Compound-boundary awareness:** If the word is in `validWords` it's already skipped at step 2. For compound words that legitimately have triple letters at morpheme boundaries (theoretically possible but written with elision in Norwegian), the decomposition engine already handles this. No special decomposition logic needed here — if the word is known, skip; if unknown and has triple letter that collapses to a known word, flag.
- **explain:** `{ nb: "Trippel bokstav — prøv ...", nn: "Trippel bokstav — prøv ..." }`

Create fixtures in `fixtures/nb/nb-triple-letter.jsonl`:
- Positive (should flag): "tykkkjer" (-> tykkjer), "skrivvver" (-> skrivver... wait, that's not valid either. Better: use realistic student typos)
- Better positives: "tykkkjer" (-> tykkjer if valid), "huuus" (-> huus -> not valid... "hus" needs double collapse? No — only collapse triple to double. "huuus" -> "huus" which isn't valid. "husss" -> "huss" not valid. Need words where triple->double = valid.)
- Most realistic: students accidentally hold a key. "skjønnnne" (-> skjønnne -> not right). Actually the regex collapses ALL triple+ to double: "tykkkjer" -> "tykkjer", "bakkke" -> "bakke", "fulll" -> "full", "villlle" -> "ville"
- Positive: "bakkke" (-> bakke), "fulll" (-> full), "villlle" (-> ville), "alllle" (-> alle), "stilllt" (-> stilt? no. "stillt" -> not valid. Try: "finnne" -> "finne"), "finnne" (-> finne)
- Negative (should NOT flag): "bakke" (valid, no triple), "stille" (valid, no triple), any word in validWords with triple letters (none expected in Norwegian)
  </action>
