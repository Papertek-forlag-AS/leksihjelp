## Integration Gap — SC-01 Zipf Tiebreaker Dead in Browser

**Location:** `extension/content/spell-check.js:207-213` (`runCheck` function)

**Defect:** The vocab object passed to `CORE.check()` contains only five fields:

```javascript
const vocab = {
  nounGenus:      VOCAB.getNounGenus(),
  verbInfinitive: VOCAB.getVerbInfinitive(),
  validWords:     VOCAB.getValidWords(),
  typoFix:        VOCAB.getTypoFix(),
  compoundNouns:  VOCAB.getCompoundNouns(),
};
```

There is no `freq` field. But `extension/content/spell-rules/nb-typo-fuzzy.js:66` reads `vocab.freq.get(cand)` for its Zipf tiebreaker:

```javascript
if (vocab && vocab.freq) {          // null guard — silently no-ops
  const z = vocab.freq.get(cand);
  if (typeof z === 'number') s += z * ZIPF_MULT;
}
```

**Why the fixture runner passed:** `scripts/check-fixtures.js` passes the full `buildIndexes()` output as `vocab`, which includes `freq: freqMap` (13,132 NB entries). The harness exercised the Zipf tiebreaker correctly — in isolation. The browser never hit the code path.

**Why Phase 3 verification missed it:** The plan-level evidence cited `vocab.freq.get(cand)` existing in the rule file and `VOCAB.getFrequency()` returning real values from the seam; both are true. Neither source-of-truth asked whether `spell-check.js` actually bridged the two. The seam exposes `getFrequency(word)` (scalar lookup) but no `getFreqMap()`.

**Fix scope:** Small. Either:
- **(a)** Add `getFreqMap: () => state.freq` to the `__lexiVocab` public surface in `vocab-seam.js` and include `freq: VOCAB.getFreqMap()` in the runCheck vocab object.
- **(b)** Rewrite the rule's scoreCandidate to call `VOCAB.getFrequency(cand)` via closure instead of reading the raw Map. Requires the rule to accept a closure instead of a vocab.freq Map — bigger refactor.

Option (a) is ~5 lines across two files and preserves the fixture/browser symmetry.

**Other E2E flows verified complete by integration checker:**
- Type a typo → spell-check popover fires (without Zipf tiebreaker — still works, just sub-optimally ranked)
- Type a prefix → word-prediction dropdown shows frequency-ranked top-3 (WP-01/03/04 wiring end-to-end)
- Language change → vocab reloads for both consumers (LANGUAGE_CHANGED broadcast, onReady re-registration)
- Fixture regression gate exercises all 5 Phase 3 rule files correctly via `readdirSync` + registry auto-discovery
