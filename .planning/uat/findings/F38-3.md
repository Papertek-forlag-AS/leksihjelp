# F38-3: FR rules silent on obvious typos — `findingsCount: 0` on "Je ma pelle Frank"

## Reproduction

1. Sideload extension v2.9.19 from source. Set popup foreign-language: French. NB locale.
2. In a French input, type:
   `Pendant que je marchais, j'ai vu un chien. Je suis Francais. Je ma pelle Frank.`

**Observed output:**

```
[lexi-spell] check {lang: 'fr', vocabSize: 18917, textHead: '...', findingsCount: 0}
```

No dots anywhere on screen. No popovers offered for "Je ma pelle" (should be "Je m'appelle" — fr-elision territory) or "Francais" (missing cedilla — should be "Français").

**Expected output:**

```
- "Je ma pelle" → fr-elision rule fires, suggests "Je m'appelle" or similar
- "Francais" → typo or accent rule fires, suggests "Français"
- findingsCount > 0 for a sentence this riddled with typos
```

## Root cause hypothesis

Same root cause family as F38-1: the API/IDB-served FR vocab is missing data the
rules depend on. With `findingsCount: 0` on text where multiple rules should fire,
the data hydration appears systematically incomplete — not just `frAspectAdverbs`.

The 4 sidecar 404s (`bigrams-fr.json`, `pitfalls-fr.json`, `freq-fr.json`, plus
`pitfalls-nb.json`) point at a sidecar-pipeline gap. Some FR rules may need
`pitfalls-fr.json` to surface common spelling slips like "Francais" → "Français".

**Confirmation needed:** check whether `validWords: 18917` actually contains
the FR base lexicon, or whether the count is misleading (e.g. it's counting
verb-conjugation forms only).

## Investigation v2 (2026-05-01, Plan 38-01.2 Task 1)

Triaged each silent rule on the test sentence:

1. **`Pendant que je marchais, j'ai vu un chien.`** — fr-aspect-hint silent.
   ROOT CAUSE: SAME as F38-1 (Papertek API serves FR generalbank from
   `lexicon/fr/` missing the 3 aspect meta entries; bundled fr.json has them
   but `BUNDLED_LANGS = ['nb']` forces FR through API path). CLOSED via the
   F38-1 v2 seam-side defensive backfill (Plan 38-01.2 Task 2). Regression
   artifact: `scripts/check-fr-bundle-completeness.js`.

2. **`Je ma pelle Frank`** — fr-elision correctly silent. `ma` is a possessive
   determiner, NOT in `ELISION_MAP` (which contains only the 9 clitics:
   je/me/te/se/le/la/de/ne/que). The bug here is a TYPO on `pelle` (for
   `appelle`), not a missed elision. fr-elision behaving as designed.
   The intended rule for this case is `fr-typo-fuzzy` (or similar typo
   coverage), but FR `typobank` is empty (0 keys in BOTH bundled fr.json
   AND API response — FR typo data has never been authored upstream).

3. **`Je suis Francais`** — accent typo (missing cedilla, `Français`).
   No FR rule currently covers diacritic-typos (cedilla, accent). FR
   `typobank` empty per (2) above.

**Outcome:** F38-3's silent-rules symptom decomposes into two distinct
classes:

- **Aspect-hint silence** (subset shared with F38-1) → CLOSED by Task 2 fix.
- **FR typo coverage gap** (NEW finding class — `pelle`, `Francais`) →
