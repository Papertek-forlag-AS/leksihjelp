# F38-1: fr-aspect-hint silent in real Chrome — French input scored against NB dictionary, FR sidecars 404

## Reproduction

1. Set popup foreign-language to French. Set NB locale.
2. Reload extension at `chrome://extensions` (reload_ts 2026-05-01T17:00:00+02:00; ext_version 2.9.18; Chrome 147.0.7727.117 arm64).
3. In a French-marked text input on any page, type: `Pendant que je marchais, j'ai vu un chien.`

**Observed output:**

```
- No spell-check dot under `marchais` (fr-aspect-hint never fires).
- No "Lær mer" panel (Step 4 not reachable since Step 1 ❌).
- `j'ai` is flagged by NB spell-check: popover reads
    "Jai står ikke i ordboken, kanskje du mente j'aime?"
  → engine is treating French tokens as Norwegian.
- Console:
    [lexi-spell] markers rendered Object
    chrome-extension://kciankappkjgeijgopddopaepjfgfknc/data/freq-fr.json:1
        Failed to load resource: net::ERR_FILE_NOT_FOUND
    chrome-extension://kciankappkjgeijgopddopaepjfgfknc/data/pitfalls-fr.json:1
        Failed to load resource: net::ERR_FILE_NOT_FOUND
    chrome-extension://kciankappkjgeijgopddopaepjfgfknc/data/bigrams-fr.json:1
        Failed to load resource: net::ERR_FILE_NOT_FOUND
```

**Expected output:**

```
- Soft-hint dot under `marchais` (or `j'ai vu`) per fr-aspect-hint rule.
- Click dot → popover with "Lær mer".
- "Lær mer" → pedagogy panel renders examples + illustration.
- French tokens treated as French; NB spell-check does NOT flag `j'ai`.
```

## Root cause hypothesis

Two-part defect, almost certainly the same root cause:

1. **Missing FR sidecar files.** `extension/data/` ships only `fr.json` + `grammarfeatures-fr.json` — `freq-fr.json`, `pitfalls-fr.json`, `bigrams-fr.json` don't exist in the bundle. `extension/content/vocab-seam.js:144-157` handles a 404 gracefully (returns `null`), so the missing files are not the proximate cause of "rule silent" — but they are evidence that FR data hydration is incomplete.

2. **Probable proximate cause: language detection.** The console fires `[lexi-spell] markers rendered`, NB spell-check fires on `j'ai` ("Jai står ikke i ordboken"), and fr-aspect-hint stays silent. This pattern matches: the engine ran spell-check against the input but treated it as Norwegian, so French rules never had a chance to evaluate. Either the per-input language plumbing isn't propagating the popup's foreign-language selection to the spell-check seam, OR the spell-check seam ignores `lang` and runs all enabled language rules (with NB winning by ordering / priority).

**Why slipped past gates:** Every release gate runs in Node against the fixture runner with `() => true` for feature predicates and synthetic input. None of the 14 gates exercise the browser's per-input language detection path. `check-vocab-seam-coverage` (Phase 38 FIX-03 anchor) defends `buildIndexes` shape, not language routing. This is a Node-vs-browser divergence, exactly the class STATE v3.2 entry called out as Pitfall 2.

Also: bigrams/freq/pitfalls sidecars for FR (and DE/ES) appear to never have been built — this is a separate but related data-pipeline gap. `npm run sync-vocab` likely doesn't generate them.

**Confirmed in Step 6 walk:** `pitfalls-nb.json` is ALSO missing (`vocab-seam.js:147 GET chrome-extension://.../data/pitfalls-nb.json net::ERR_FILE_NOT_FOUND`). NB still works because `loadBundledSidecar` returns `null` on 404 and the rule path doesn't depend on it for the gates the walker tested. So the pipeline gap is broader than FR — it's universal. The fix should regenerate sidecars for all languages, not just FR.
