# F38-5: fr-aspect-hint over-fires on narrative structures (background imparfait + punctual passé composé)

## Reproduction

1. Sideload extension v2.9.27. Set popup foreign-language: French. NB locale.
2. Type the canonical narrative sentence:

```
Pendant que je marchais, j'ai vu un chien.
```

**Observed output:**

- fr-aspect-hint fires on `j'ai vu`.
- Tooltip: "j'ai vu (passé composé) — adverb «Pendant que» cues imparfait".
- Popover offers "Lær mer" with passé-composé vs imparfait pedagogy.

**Expected output:**

```
- No marker. The sentence is grammatically correct French.
```

## Why it's a false positive

This is canonical **narrative French**:
- `Pendant que je marchais` — subordinate clause, imparfait ✓ correct (cued by `pendant que`)
- `j'ai vu un chien` — main clause, passé composé ✓ correct (one-shot event interrupting the background)

The "background imparfait + punctual passé composé" pattern is textbook
narrative structure (e.g. "While I was walking, I saw a dog"). The rule
should flag intra-clause aspect mismatches like:

- ❌ `Hier je mangeais une pomme` (hier cues PC, mangeais is imparfait)
- ❌ `Tous les jours j'ai mangé une pomme` (tous les jours cues imparfait, j'ai mangé is PC)

But not cross-clause structures where each clause has the correct aspect
relative to its OWN cue.

## Root cause hypothesis

`fr-aspect-hint.js` `check()` operates on the full sentence token range
(line ~140) without tracking clause boundaries. When it finds an adverb
hit (e.g. `pendant que` at index 0-1), it scans the entire sentence for
verbs in the "wrong" aspect — but a comma + new clause introduces a new
verb-aspect contract that the original cue doesn't govern.

Pedagogically, the rule is most useful for *intra-clause* mismatches.
Cross-clause contrast (background imparfait + punctual passé composé) is
exactly the pattern the language is designed to express; flagging it is
counterproductive.

## Fix candidates (deferred to future plan)

1. **Clause splitting:** Split sentence tokens on commas, conjunctions,
   `et`, `mais`, `ou`. Run the adverb scan per clause. Verb-aspect check
   only fires within the clause containing the adverb hit.
2. **Adverb-specific scope:** Some imparfait adverbs (e.g. `pendant que`,
   `tandis que`, `quand j'étais`) introduce a subordinate clause whose
   aspect contract doesn't extend to the main clause. Encode this in the
   adverb data — `scope: 'subordinate-only'` — and skip cross-clause
   checks for those adverbs specifically.
3. **Surface as a softer suggestion:** Instead of P3-hint with a flag,
   surface as a "consider the aspect" prompt that's clearly informational
   rather than corrective. (Possibly redundant if (1) or (2) lands.)

Recommend approach (2) — narrower scope, smaller change, addresses the
common false-positive class. Approach (1) is more general but riskier.

## Why this isn't blocking

- Rule still fires correctly on intra-clause mismatches (`Hier je
  mangeais une pomme` — confirmed in walker re-walk Step 1).
- "Fiks" button correctly hidden (F38-4 follow-up commit `6a6ae64`)
  because `finding.fix === finding.original` for hint findings — student
  can dismiss with Avvis without infinite loop.
- Lær mer pedagogy still teaches the aspect distinction, which is
  educationally valuable even on borderline cases — student learns the
  framework even if THIS sentence happens to be a correct narrative.
- Saturday friend testing on stb-lockdown.app: friends writing in NB
  won't hit it; only French-writing testers might encounter it on
  multi-clause sentences with comma-separated narrative structures.
