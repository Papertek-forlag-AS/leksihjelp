## Integration Gaps

### EXAM-08 — Lockdown cross-app sync incomplete

The lockdown sibling project (`/Users/geirforbord/Papertek/lockdown`, deployed to stb-lockdown.app / papertek.app) consumes leksihjelp content scripts via `node scripts/sync-leksihjelp.js`. Phase 27 introduced exam-mode but the sync pipeline was not extended:

1. **`extension/exam-registry.js` is not copied.** It lives outside the four trees the sync script walks (`content/`, `data/`, `styles/`, `i18n/`). Result: `host.__lexiExamRegistry` is undefined in lockdown → `__lexiExam.isSurfaceSafe(...)` falls through to fail-safe "unknown surface = hidden" branch (`spell-check-core.js:488`). Behaviour is over-restrictive rather than dangerous, but inconsistent with extension behaviour.
2. **Loader does not inject exam-registry.js.** `lockdown/public/js/leksihjelp-loader.js` `LEKSI_BUNDLE` array (line ~372) does not include `leksihjelp/exam-registry.js`. Even after a re-sync that copies the file, the loader will not load it.
3. **Synced content scripts are stale.** mtime on lockdown's `floating-widget.js`, `word-prediction.js`, `spell-check.js` is 2026-04-27 (predates Phase 27 commits 9e99357, d02cc24, 3b9dc25, etc.). `grep examMode` in lockdown's synced files returns 0 hits.
4. **Lockdown teacher-lock writer not implemented.** `examModeLocked` is consumed by `extension/popup/popup.js:2436-2467`, but no code in lockdown writes `chrome.storage.local.examModeLocked` or `examMode`. The documented teacher-control mechanism is non-functional on lockdown until this is wired.
