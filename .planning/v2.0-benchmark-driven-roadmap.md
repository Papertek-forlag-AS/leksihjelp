# v2.0 Benchmark-Driven Roadmap (Draft)

**Status:** DRAFT — not yet a committed milestone. Run
`/gsd:new-milestone` with this doc as the seed when ready to formalize.

**Theme:** "Depth of coverage — grammar governance beyond tokens."
v1.0 shipped per-token rules (typo, gender, särskriving, modal form,
homophones, dialect-mix). v2.0 pushes into *structural* errors: word
order, case governance, agreement spanning clauses, aspect/mood
choice, register drift. Everything in this doc was surfaced by the
`benchmark-texts/*.txt` passages as "FUTURE / UNPLANNED" on
2026-04-24.

**Source of truth for validation:** `benchmark-texts/<lang>.txt`.
Every phase below names specific lines in specific benchmark files
that should flip from unflagged → flagged when the phase ships.
If a phase lands and the relevant benchmark line still shows no
underline, the phase did not achieve its goal regardless of fixture
green.

**Cross-cutting constraints (inherited from v1.0, don't violate):**
- SC-06 network silence — spell-check surface stays offline.
- Bundle size stays under 20 MiB internal cap.
- Data-logic separation — new pattern lists live in
  `papertek-vocabulary`; rule code stays in this repo.
- `check-fixtures` hard gates every release. New rules ship with
  ≥30 positive + ≥15 acceptance fixture cases per language.
- Every popover-surfacing rule needs an `explain()` contract and a
  CSS dot-colour binding — paired self-tests enforce this.

---

## Phase groupings

Ordered by student-impact × implementation tractability. Rough
effort bands: **S** = days, **M** = 1–2 weeks, **L** = 2–4 weeks.
Dependencies flow top-to-bottom; later phases assume earlier ones
landed. Parallel tracks where marked.

### Phase 6 — Register & stylistic polish [S, cross-lang]

Low-risk, heuristic-friendly warnings. Good opening phase because
it reuses existing lookup infrastructure and ships UX value fast.

- **6.1 Register/formality detector.** Flag `gonna`, `wanna`,
  `ain't`, `kinda` in EN; flag anglicisms `downloade`, `booket`
  in NB; flag colloquial negation `je sais pas` in FR when
  surrounding text is formal. Register inferred from sentence
  length / capitalization / punctuation density (heuristic, not
  ML). Opt-in via grammar feature toggle.
- **6.2 Collocation errors (EN seed).** Curated "wrong-verb"
  collocation list in `papertek-vocabulary`: `make a photo → take`,
  `big rain → heavy`, `strong knowledge → deep/solid`. Data-only;
  rule is a plain bigram lookup.
- **6.3 Stylistic redundancy.** Small list: `return back`, `free
  gift`, `future plans`, `past history`. One phrase-bank file per
  language; rule is a literal match.
- **Validates against:** `en.txt` lines with `gonna`, `return back`;
  `nb.txt` anglicism leakage.

### Phase 7 — Word-order violations [M, NB+DE+FR]

Structural rule. Harder than Phase 6 — needs light syntactic
reasoning (finite-verb position, adverbial fronting).

- **7.1 NB V2.** Flag `<fronted-adv> <subject> <verb>` in main
  clauses ("I går jeg gikk"). Detection: time/place adverbial at
  position 0 followed by noun/pronoun followed by finite verb.
  Acceptance cases: interrogatives, subordinate clauses, V2-compliant
  inversions. High false-positive risk — budget generous fixtures.
- **7.2 DE V2 + subordinate verb-final.** Two sub-rules sharing
  detection code: main-clause V2 ("ich bin gegangen zu …") and
  subordinate-clause verb-final ("dass er ist nett" should be
  "dass er nett ist"). Trigger subordinator-set `dass/weil/wenn/ob/…`.
- **7.3 FR BAGS adjective placement.** Beauty/Age/Goodness/Size
  adjectives go before noun; most others after. Closed-set list
  (~40 adjectives) in `papertek-vocabulary`.
- **Validates against:** `nb.txt` "Hvorfor du tror";
  `de.txt` "Letzte montag ich bin gegangen", "dass er ist nett";
  `fr.txt` "une belle femme" vs post-noun defaults.

### Phase 8 — DE case & agreement governance [M, DE-focused]

High error-density for Norwegian learners. Closed morphology,
tractable lookup.

- **8.1 Preposition–case governance.** Preposition table with
  required case per prep; two-way preps (`in/an/auf/über/unter/…`)
  flag a warning note "check motion vs location" rather than hard
  reject. Fixture emphasis on dative/accusative mismatch
  ("mit dem" correct, "mit den" wrong for singular).
- **8.2 Trennbare Verben split.** Detect separable-prefix verbs
  (`aufstehen`, `mitkommen`, …) used in main clauses without
  splitting ("ich aufstehe" → "ich stehe auf"). List lives in
  `papertek-vocabulary` verbbank with `separable: true`.
- **8.3 Perfekt auxiliary choice.** Movement/state-change verbs
  take `sein`, others take `haben`. Curated `aux: "sein"` flag on
  verb entries.
- **8.4 Noun-compound gender from last component.** When a noun
  doesn't exist in the bank as-is, split on compound boundaries
  and assign gender from the final component. "Schultasche" →
  gender from "Tasche" (f).
- **Validates against:** most of `de.txt`.

### Phase 9 — ES ser/estar, por/para, personal "a" [M, ES-focused]

Closed-trigger distinctions. Paper-tractable.

- **9.1 ser vs estar.** Copula disambiguation by predicate type:
  profession/nationality/permanent → `ser`; location/temporary
  state/progressive → `estar`. Predicate adjective list in
  `papertek-vocabulary` tags each as `copula: "ser"|"estar"|"both"`.
- **9.2 por vs para.** Preposition contrast rule with ~15 trigger
  patterns (duration → `por`, deadline → `para`, cause → `por`,
  purpose → `para`, …). Rule wraps a decision tree.
- **9.3 Personal "a".** Flag human direct objects missing
  preceding `a` ("veo Juan" → "veo a Juan"). Requires knowing
  which nouns are human — seeded in papertek with `human: true`.
- **Validates against:** `es.txt` "Soy cansado", "Veo Juan",
  "para comprar comida por mi familia".

### Phase 10 — FR élision, auxiliary, participe passé [M, FR-focused]

- **10.1 Élision before vowel.** `le/la/je/que/si/ne` + vowel-initial
  word → apostrophe form. Closed and deterministic.
- **10.2 être vs avoir auxiliary.** DR MRS VANDERTRAMP set +
  pronominal verbs take `être`. Data field on verbbank.
- **10.3 Participe passé agreement with preceding DO.** The hard
  one. Detect `avoir` + past participle with a preceding direct-
  object pronoun (`la`/`les`/`que`); require participle to agree.
  Known corner cases — ship with explicit "grammar_fr_pp_agreement"
  feature toggle so students can disable.
- **Validates against:** `fr.txt` "à le magasin", "je ai",
  "si il pleut", "la pomme que j'ai mangée".

### Phase 11 — Aspect & mood [L, ES+FR, parallel to 9/10]

Depends on basic trigger detection (from Phases 9 and 10).

- **11.1 ES subjuntivo triggers.** After `quiero que`, `espero que`,
  `dudo que`, `es importante que`, etc., the embedded verb must
  be subjunctive. Trigger list closed; verb inflection check uses
  existing conjugation data.
- **11.2 ES pretérito vs imperfecto.** Aspectual hints: `ayer`,
  `la semana pasada` → pretérito; `mientras`, `siempre`, `cada día`
  → imperfecto. Warn, don't hard-flag — aspect can legitimately
  vary.
- **11.3 FR subjonctif triggers.** After `il faut que`, `avant que`,
  `bien que`, `pour que`. Mirror of 11.1.
- **Validates against:** `es.txt` "Quiero que mi hermano viene";
  `fr.txt` "Il faut que je parle".

### Phase 12 — Pronoun & pro-drop [M, ES+FR]

- **12.1 ES pro-drop overuse.** Flag redundant subject pronouns
  when verb agreement disambiguates ("yo voy a la playa" →
  "voy a la playa"). Soft warning.
- **12.2 ES gustar-class syntax.** Detect `sujeto + gustar-verb +
  objeto` pattern ("yo gusto la pizza") and suggest dative
  restructuring ("me gusta la pizza").
- **12.3 FR double-pronoun order.** Clitic cluster order
  me/te/se/nous/vous < le/la/les < lui/leur < y < en. Cluster
  detection + permutation check.
- **Validates against:** `es.txt` "Él no gusta ayudar",
  "Yo pienso", "yo voy a la playa".

### Phase 13 — Register consistency within a text [L, cross-lang]

Stateful across sentences. Hardest of the soft-warning phases
because it requires a document-level analysis pass.

- **13.1 DE du/Sie drift.** Single-text consistency check.
- **13.2 FR tu/vous drift.** Mirror of 13.1.
- **13.3 NB bokmål/riksmål mixing.** Flag `boken`/`efter`/`sne`
  alongside standard bokmål forms in the same paragraph.
- **13.4 NN a-infinitiv / e-infinitiv mixing.** NN tolerates both
  but not in the same register-text.
- **Validates against:** `nb.txt` "boka"+"boken", `nn.txt`
  mixed-infinitive paragraphs, `de.txt`/`fr.txt` address-form drift.

### Phase 14 — Morphology & agreement beyond tokens [M, EN+ES+FR]

- **14.1 EN morphological overgeneration.** `goed/runned/childs/
  mouses/womans`. Curated irregular list — any form matching
  `<regular-pattern>` for a verb/noun on the irregular list gets
  flagged.
- **14.2 ES/FR opaque-noun gender.** For nouns whose gender can't
  be inferred from ending, rule relies on `genus` field in
  vocabbank (already present). New rule: when article + noun
  mismatch, flag. Requires article disambiguation.
- **14.3 EN word-family confusion.** `creative/creativity/creation`,
  `succeed/success/successful`. Closed word-family list in
  papertek; rule fires when POS mismatches surrounding syntax.

### Phase 15 — Collocations & idioms at scale [L, cross-lang]

Depends on Phase 6.2 as proof of pattern. Turns the curated
seed-list into a systematic per-language collocation bank.

- **15.1 NB preposition collocations.** `flink til`, `glad i`,
  `bra med`. List-driven.
- **15.2 DE/FR/ES preposition governance.** Same shape.
- **15.3 Idiomatic literalism detection.** "Det regner katter og
  hunder" → suggest idiomatic alternative. Risky; scope TBD.

### Phase 16 — Tense harmony & discourse [L, aspirational]

Cross-sentence coherence. Pushes into light NLP territory —
hold as last phase; may get deferred to v3.0.

- **16.1 Unmotivated past→present switches.** Tag verbs as past/
  present; flag runs where tense flips without discourse markers.
- **16.2 Anaphora ambiguity (NB first).** Pronoun referent
  unclear when two candidates of matching gender exist in recent
  context. Soft warning.
- **16.3 Long-distance subject–verb agreement.** Subject and verb
  separated by embedded clause; check agreement.

## Sequencing notes

- **Parallel tracks possible:** Phase 6 can ship alongside Phase 7.
  Phases 8 / 9 / 10 are language-siloed and independent.
- **Gate before Phase 11:** confirm Phase 9/10 trigger-detection
  infrastructure can be reused — don't duplicate.
- **Phase 13 depends on** document-level state management that no
  current rule needs. Plan a seam change in Phase 13's research
  step.
- **Phase 16 is a milestone boundary candidate** — if effort
  estimates blow up, split into v3.0.

## Non-goals for v2.0

- Machine-learning rules. Every phase above is deterministic or
  lookup-driven.
- Runtime network calls. SC-06 holds.
- Changing the storage model (Firestore schema, vocab API shape).
  Data enrichment only.
- Dialect-specific rules beyond NB/NN (no nordlandsk, no sunnmørsk).

## Validation protocol

When a phase closes, re-run the relevant `benchmark-texts/<lang>.txt`
through the extension manually and count: of the lines the phase
promised to cover, how many now flag? Target 80% of promised lines
before closing the phase; unflagged remainder becomes a carry-over
fixture for the next cycle.
