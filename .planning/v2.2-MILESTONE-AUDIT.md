# v2.2 Milestone Audit: Student Language Intelligence

**Audited:** 2026-04-26T20:00:00Z
**Status:** GAPS FOUND
**Score:** 2/12 requirements satisfied

## Executive Summary

Milestone v2.2 has critical gaps. Phase 21 (Dictionary Intelligence) shipped rendering functions and data, but a **data pipeline break** means false-friend warnings and sense-grouped translations never actually reach the user. Phase 22 (å/og Confusion) was never started — all 4 AAOG requirements are orphaned.

### Root Cause: Data-Renderer Mismatch

The `falseFriends` and `senses` data lives on **nb.json** (Norwegian) entries. But both popup.js and floating-widget.js render results from **foreign-language** entries (de/es/fr.json), which have no enrichment fields. The rendering functions exist, are correctly placed in the template, and pass static code inspection — but they receive entries without the data they need and produce empty output for all user-facing scenarios.

## Integration Findings

### BLOCKER: falseFriends data pipeline broken (FF-01, FF-03, FF-04)

**Break point:** `popup.js` search flow pushes foreign-language entries to results (line 1098-1104). `renderFalseFriends(entry)` at line 1309 receives these foreign entries, which have no `falseFriends` field. The function returns `''` for all scenarios.

In `floating-widget.js`, the widget loads `currentLang.json` (de/es/fr), never `nb.json`. `match.falseFriends` is always `undefined`.

**Secondary issue:** Even in monolingual NB mode where nb entries ARE in `allWords`, the filter `f.lang === currentLang` tests `f.lang === 'nb'`, which never matches stored values (`'en'`, `'de'`, `'es'`, `'fr'`).

### BLOCKER: senses data pipeline broken (POLY-01, POLY-03, POLY-04)

**Same root cause.** `renderSenses(entry)` at line 1310 receives foreign entries with no `senses` field. Returns `null`, falls through to flat translation display.

The `senses` data on nb.json `på_contr` entry has translations keyed by target language (`de`, `en`, `es`, `fr`), but in NB mode the filter `s.translations[currentLang]` tests for `'nb'` key which doesn't exist.

### Fix Options

1. **Mirror enrichment at sync time:** `sync-vocab.js` copies `falseFriends`/`senses` from nb entries onto corresponding foreign-language entries during the sync. Pros: no runtime cost. Cons: data duplication across files.
2. **Cross-dictionary lookup at render time:** After finding a foreign match, look up the corresponding nb entry to retrieve enrichment fields. Pros: single source of truth. Cons: requires loading nb.json in all search modes.
